## Executive summary

The Kacho control-plane polyrepo is, on the whole, a mature and security-conscious platform: zero CRITICAL findings across all 12 repos, uniformly parameterized SQL, crypto/rand IDs, fail-closed authz interceptors, sentinel-based error mapping that scrubs pgx/SQLSTATE text, DB-per-service with DB-level invariants (FK/UNIQUE/EXCLUDE/CAS), and disciplined Clean-Architecture layering with strong concurrency test coverage in the flagship services (kacho-vpc, kacho-corelib, kacho-nlb). The residual risk is concentrated and thematic rather than pervasive: the 12 HIGH findings cluster around authorization edges (List enumeration and Move/destination-project checks that skip server-side IAM Check, and one proto service — InstanceGroupService — shipping 23 public RPCs with zero authz annotations), LRO/Operation polling authz (geo fail-closes it, registry leaves it cross-tenant readable), and TOCTOU lost-update Update paths that Get-then-write instead of atomic CAS (compute Instance.Update most dangerously, since it clobbers concurrent lifecycle transitions). The worst-posture areas are the two UI codebases (dead secure-client layer so DPoP/step-up MFA are not actually enforced, known-vulnerable npm deps, no CSP headers, and 263 byte-identical duplicated files creating security-drift risk) and CI test-execution gaps (compute and registry ship concurrency/durability safety-net tests that are silently never run). Nearly all fixes are contract-safe internal changes — the only contract-touching item is adding authz annotations to InstanceGroupService.proto, which is additive and non-breaking. No frozen-contract redesign or wire-breaking change is required to close any HIGH finding.

## Systemic themes

### [HIGH] Missing or bypassable server-side authorization on read/enumeration and cross-project moves (BOLA / cross-tenant enumeration)

- **Repos:** kacho-proto, kacho-nlb, kacho-vpc, kacho-compute
- **Recommendation:** Add authz annotations to InstanceGroupService's 23 RPCs (proto) and regenerate the permission catalog. Make List RPCs enforce a server-side FGA Check / ScopeFiltered subject that cannot short-circuit to BypassAll for a system/empty principal (nlb, vpc). Wire a real destination-project Check into nlb Move. Do not trust client-header AssertProjectOwnership as a fallback when the list-filter is disabled — that is the shipped helm default and must fail closed.

### [HIGH] LRO / OperationService authorization inconsistency

- **Repos:** kacho-geo, kacho-registry, kacho-compute
- **Recommendation:** Standardize the Operation poll surface across services: register OperationService.Get/Cancel in every PermissionMap so the fail-closed interceptor does not deny normal LRO polling (geo), and use the owner-scoped GetOwned/CancelOwned from corelib instead of the unscoped Get/Cancel so operations are not cross-tenant readable/cancelable by id (registry). Add a shared conformance test asserting the poll flow both works for the owner and denies others.

### [HIGH] TOCTOU / lost-update on Update paths (Get-then-full-row-UPDATE instead of atomic CAS)

- **Repos:** kacho-compute, kacho-vpc, kacho-iam, kacho-api-gateway
- **Recommendation:** Convert remaining read-modify-write Update paths to atomic single-statement CAS / SELECT FOR UPDATE / xmin-OCC per project rule #10. Priority: compute Instance.Update (writes stale status column, clobbering the SetStatusCAS lifecycle machine) and vpc Address/NetworkInterface/AddressPool (left on plain Get+UPDATE while the other five resources were fixed — deletion_protection can silently revert). Also close the api-gateway idempotency-store check-then-act gap and the ConditionsService delete refcheck in iam. Add exactly-one-winner concurrent regression tests for each.

### [HIGH] Concurrency/durability safety-net tests exist but never execute in CI

- **Repos:** kacho-compute, kacho-registry
- **Recommendation:** Fix CI job targeting so internal/clients integration + two-replica exactly-once/drainer tests actually run (compute integration job only targets ./internal/repo/..., unit job runs -short, so those tests are dead). Backfill a test for the registry FGA register-drainer applier. A safety net that never runs is worse than none because it implies coverage that is absent.

### [HIGH] Transactional-outbox consumer integrity gap

- **Repos:** kacho-registry
- **Recommendation:** Fix the register-drainer applier so ErrAlreadyApplied on one tuple does not short-circuit the remaining tuples in a multi-tuple Create/RepoPush intent on retry-after-partial-apply. Iterate all tuples idempotently; the outbox's 'owner-tuple is never lost' guarantee currently breaks under partial-apply retry. Corelib's own drainer is correct — align the consumer to it.

### [HIGH] Client-side security controls implemented but not actually enforced on real traffic (dead secure path)

- **Repos:** kacho-ui, kacho-api-gateway, kacho-ui-future
- **Recommendation:** Route real API traffic through the DPoP + bearer-token + per-request step-up client instead of the parallel plain fetch client (kacho-ui's 50 resource modules bypass it entirely), and wire the gateway step-up/ACR gate (PermissionLookup unwired, REST->FQN mapper broken) so MFA enforcement does not depend solely on the IAM FGA model. Add tests that assert an unbound/step-up-required call is rejected.

### [HIGH] Frontend supply-chain and HTTP-hardening gaps

- **Repos:** kacho-ui, kacho-ui-future, kacho-deploy
- **Recommendation:** Upgrade known-vulnerable production deps (react-router-dom, monaco-editor->dompurify) and add CSP / anti-clickjacking / hardening response headers to the nginx/ingress config for the SPAs. Pair with deploy-repo hardening: namespace PSA enforcement, per-container securityContext on kacho-iam workloads, immutable digest-pinned images (no :latest), and a CI image/dependency vuln-scan gate.

### [HIGH] Massive copy-paste duplication in the micro-frontend shells (security-drift risk)

- **Repos:** kacho-ui-future, kacho-ui
- **Recommendation:** Extract the byte-identical duplicated code (263 files across vpc/iam remotes, including the 2845-line resource-registry.tsx and the security-critical dpop/api-client/config modules) into a shared package. Duplicated security code drifts independently and defeats single-point-of-fix; violates the corelib-reuse principle.

### [HIGH] Security-control code paths lacking negative/behavioral tests

- **Repos:** kacho-corelib
- **Recommendation:** Add negative tests for the breakglass anonymous-bypass guard (isAnonymousSubject closed-list) and coverage for ListenInvalidator revocation propagation, the Check-cache concurrency path, and the rate-limiter refill/eviction — all security- or DoS-relevant and currently untested in the platform's authz engine library.

### [LOW] Config-convention and cross-cutting-primitive duplication (deliberate/low-risk, track as tech-debt)

- **Repos:** kacho-corelib, kacho-api-gateway, kacho-geo, kacho-registry, kacho-nlb
- **Recommendation:** Note but do not block on: envconfig struct-tags vs the evgeniy YAML/viper rule is a uniform platform-wide corelib convention (deliberate). Genuinely worth consolidating: the TTL+LRU cache hand-rolled six times in api-gateway, the god-files (authz.go 1269 lines, resource-registry modules), and the triplicated OpenFGA authz vocabulary in registry — centralize to reduce authz-drift risk.


## Prioritized remediation

1. **[high]** Add authz annotations to InstanceGroupService (23 public RPCs, zero authz gate) and regenerate permission catalog  _(contract_safe=true)_
   - repos: kacho-proto, kacho-api-gateway
   - Broadest missing-authorization/BOLA surface: 23 public RPCs including access-binding mutation with no authz annotation feeding the gateway middleware. Blast radius depends on gateway fail-open-vs-closed default. Adding annotations is additive/non-breaking on the wire.

2. **[high]** Close cross-tenant List enumeration and unauthorized Move: enforce server-side FGA Check on List (no BypassAll for system/empty subject) and add destination-project authz to nlb Move  _(contract_safe=true)_
   - repos: kacho-nlb, kacho-vpc
   - Two live cross-tenant authorization holes: List RPCs short-circuit to BypassAll for a system/empty principal enabling resource enumeration (nlb + vpc NetworkService/List falling back to client-header-trusted auth, which is the shipped helm default), and Move authorizes only the source, not the destination project. Internal fixes; contracts unchanged.

3. **[high]** Fix compute Instance.Update to CAS the status column instead of stale full-row write, and convert vpc Address/NIC/AddressPool Updates to atomic CAS  _(contract_safe=true)_
   - repos: kacho-compute, kacho-vpc
   - Instance.Update writes a stale status in a full-row UPDATE, silently reverting concurrently-committed lifecycle transitions and defeating the SetStatusCAS state machine — real data-corruption/second-writer-wins. The same plain-Get+UPDATE pattern remains on three vpc resources (deletion_protection can silently revert). Ship with exactly-one-winner concurrent regression tests per rule #10/#12.

4. **[high]** Fix registry register-drainer applier to not drop later tuples on retry-after-partial-apply (outbox integrity)  _(contract_safe=true)_
   - repos: kacho-registry
   - ErrAlreadyApplied short-circuits the per-tuple loop, silently dropping the owner-tuple of multi-tuple Create/RepoPush intents on retry — breaks the outbox at-least-once 'owner-tuple never lost' guarantee, causing missing FGA authz materialization (a downstream authz-correctness failure). Add the currently-absent applier test.

5. **[high]** Standardize OperationService/LRO authz: register Get/Cancel in geo PermissionMap and switch registry to owner-scoped GetOwned/CancelOwned  _(contract_safe=true)_
   - repos: kacho-geo, kacho-registry
   - geo omits OperationService.Get/Cancel from PermissionMap so the fail-closed interceptor denies all LRO polling under the default secure config (availability/functional break, safe direction). registry uses the unscoped Get/Cancel so operations are cross-tenant readable/cancelable by id (real, if id-gated, cross-tenant read). Both are small, contract-safe corrections; add a shared conformance test.

6. **[high]** Wire the dead integration/concurrency safety-net tests into CI  _(contract_safe=true)_
   - repos: kacho-compute, kacho-registry
   - compute's internal/clients FGA-drainer exactly-once + two-replica race tests never execute (integration job targets only ./internal/repo/..., unit job runs -short); the registry applier is untested. These guard the very integrity paths flagged above, so they must run before those fixes can be trusted. Contract-safe CI config change.

7. **[high]** Route real UI API traffic through the secure client and wire the gateway step-up/ACR gate so DPoP binding and per-request MFA are actually enforced  _(contract_safe=true)_
   - repos: kacho-ui, kacho-api-gateway, kacho-ui-future
   - An elaborate DPoP + step-up-MFA layer is fully built but dead: every resource module uses a plain fetch client with no token/DPoP/401-403 handling, and the gateway step-up gate (PermissionLookup unwired, broken REST->FQN mapper) is dead code — so MFA/token-binding is not enforced on the real API surface, leaving enforcement solely on the IAM FGA model. Internal refactor; add tests asserting step-up-required calls are rejected.

8. **[high]** Upgrade vulnerable frontend dependencies and add CSP / clickjacking / hardening headers  _(contract_safe=true)_
   - repos: kacho-ui, kacho-ui-future, kacho-deploy
   - Known-vulnerable production npm deps (react-router-dom, monaco-editor->dompurify) plus complete absence of CSP/anti-clickjacking response headers on the nginx-served SPAs. Straightforward dependency bumps + ingress/nginx config; no contract impact.

9. **[high]** Add negative tests for corelib breakglass anonymous-bypass guard and untested authz/rate-limit controls  _(contract_safe=true)_
   - repos: kacho-corelib
   - The breakglass isAnonymousSubject closed-list, ListenInvalidator revocation propagation, Check-cache concurrency, and rate-limiter refill/eviction are security/DoS-relevant controls in the platform's shared authz engine with no coverage — a regression here silently weakens every consuming service. Test-only, contract-safe.

10. **[high]** De-duplicate the vpc/iam micro-frontend shells into a shared package  _(contract_safe=true)_
   - repos: kacho-ui-future, kacho-ui
   - 263 byte-identical files (incl. the 2845-line resource-registry and the security-critical dpop/api-client/config) duplicated across remotes create security-drift risk and defeat single-point-of-fix for items 7-8. Extract to a shared package per the corelib-reuse principle. Larger internal refactor, no contract impact.

11. **[medium]** Harden the deploy manifests: namespace PSA, per-container securityContext, immutable digest-pinned images, CI vuln-scan gate  _(contract_safe=true)_
   - repos: kacho-deploy
   - Defense-in-depth infra gaps: the kacho application namespace lacks PSA enforcement, in-tree kacho-iam workloads (main + init + siem-forwarder) ship without securityContext, base values use mutable :latest, and CI has no image/dependency scan gate. No tenant-crossing defect but the highest-value infra hardening. Contract-safe.

12. **[medium]** Consolidate cross-cutting primitives and split authz god-files (tech-debt / drift reduction)  _(contract_safe=true)_
   - repos: kacho-api-gateway, kacho-registry
   - The TTL+LRU cache is hand-rolled six times in api-gateway (one diverging implementation), authz.go is a 1269-line god-file whose decide() is a 333-line branch cascade on the security-critical path, and registry triplicates its OpenFGA authz vocabulary across packages (authz-drift risk). Refactor to corelib primitives / centralized vocabulary for testability and review safety. Internal, contract-safe.
