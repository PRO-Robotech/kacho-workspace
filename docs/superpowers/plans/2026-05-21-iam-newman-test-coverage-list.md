# IAM — Newman Test Coverage List (100%)

> Полный список newman-тестов для 100% покрытия домена **IAM**.
> Источник RPC — `kacho-proto/proto/kacho/cloud/iam/v1/*.proto` (извлечено 2026-05-22).
> **25 gRPC-сервисов** + `ApiTokenService` (Блок F — новый) + `OperationService` + SCIM/SAML (REST).
>
> **Методология (CLAUDE.md §Запреты #11):** каждый кейс пишется test-first — RED до кода.
> Каждый RPC обязан иметь ≥1 happy + ≥1 negative; authz-gated — плюс authz-кейсы.
>
> **Классы кейсов:** `CRUD` happy · `NEG` negative · `AUTHZ` (grant/deny/anon/foreign) ·
> `BVA` границы полей · `IDEM` идемпотентность · `SEC` injection · `FLOW` stateful.
> **ID-формат:** `IAM-<SVC>-<RPC>-<CLASS>[-detail]`.

## 8. Сквозные flow-кейсы (authz-матрица)

Помимо per-RPC — stateful flow на authz-переходы (как `AUTHZ-REVOKE-ENFORCED`):
- `IAM-FLOW-GRANT-REVOKE` — grant→allow→revoke→deny (per resource type: account/project/group/SA).
- `IAM-FLOW-CUSTOM-ROLE-PERMISSIONS` — custom-роль с granular permissions даёт ровно свои права.
- `IAM-FLOW-JIT-ACTIVATE-EXPIRE` — JIT activate→allow→expiry→deny.
- `IAM-FLOW-BREAKGLASS-2APPROVE` — 2-person approve→cluster-admin allow.
- `IAM-FLOW-INVITE-ACCESS` — invite→invitee получает project-доступ.
- `IAM-FLOW-APITOKEN-LIFECYCLE` — issue→authn allow→revoke→authn 401.
- `IAM-AUTHZ-MATRIX` — 6 субъектов (ANON/NOB/PA1/AAA/AAB/INV) × все authz-gated RPC (расширение `authz-deny.py`).

- **Internal-сервисы** гоняются через internal-mux (port 9091 / `iam-internal-only-check.py`-паттерн), не на external TLS.
