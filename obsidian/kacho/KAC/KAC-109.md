# KAC-109: E4 — IAM UI block

**Status**: test (UI deployed на e2c825; signup-flow перенесён в follow-up под E2/E3)
**Type**: feature (sub-эпик E4 эпика [[KAC-104]])
**Repos**: kacho-ui (основная работа) + kacho-deploy (overrides.yaml) + kacho-workspace (acceptance v2 + vault)
**YT**: https://prorobotech.youtrack.cloud/issue/KAC-109

## Что и зачем

UI-block для IAM на E0 деплое e2c825 — Zitadel/OpenFGA выключены, signup-flow отложен до KAC-107 (E2) / KAC-108 (E3). В этом тикете реализована UI-часть KAC-104 DoD #2/#3/#4/#7 (CRUD из UI + реактивность + Operations principal column). Signup-flow + auth-context — выделено в follow-up.

**Что сделано**:
- **`kacho-ui`** (новая ветка `KAC-109`):
  - `src/api/iam.ts` — типизированный API client для 7 IAM сервисов (Account/Project/User/SA/Group/Role/AccessBinding) + custom verbs (Move, addMember, removeMember, listByResource, listBySubject).
  - `src/pages/iam/*.tsx` — 7 страниц-CRUD: AccountsPage, ProjectsPage, UsersPage (read+delete), ServiceAccountsPage, GroupsPage (с inline GroupMembersPanel expand-row), RolesPage (system read-only, custom editable, permissions JSON-editor с regex validation), AccessBindingsPage (Segmented byResource/bySubject + Create modal с subject+role+resource pickers).
  - `src/components/iam/IamLayout.tsx` — общая обёртка с tabs + info-banner про E0 (no auth).
  - `src/components/iam/IamCommon.tsx` — `useIamMutation` (POST/PATCH/DELETE/ACTION + Operation polling + invalidate + toast), `CopyableMonoId`, `fmtTs`, `SystemTag`.
  - `src/App.tsx` — добавлены 7 routes под `/iam/{accounts,projects,users,service-accounts,groups,roles,access-bindings}` в обёртке IamLayout.
  - `src/lib/service-modules.tsx` — IAM-кнопка в `COMMON_BOTTOM` сайдбара (`LockOutlined`).
  - `src/components/OperationsTable.tsx` — колонка «Кем создано» заменена с `created_by` на `<PrincipalCell>` (icon-по-типу `user`/`service_account`/`system`/anonymous + Tag + tooltip с id+display_name; backward-compat fallback на `created_by`).
  - `vite.config.ts` + `deploy/nginx.conf` — proxy для `/iam/v1/*` (с разделением: SPA route `/iam/accounts` через SPA fallback, API `/iam/v1/*` proxied to api-gateway).
- **`kacho-deploy`** — `clusters/e2c825/overrides.yaml` `ui.image` → `ttl.sh/kacho-kac109-202605171612/kacho-ui:24h`.
- **`kacho-workspace`** — `docs/specs/sub-phase-2.0-iam-E4-signup-flow-ui-acceptance.md` обновлён до v2 (APPROVED): 4 blocker'а адресованы (Zitadel/OpenFGA выключены → signup отложен; RTK Query → @tanstack/react-query; no /signup landing → инфо-banner; работа без default Account → empty-state UX).
