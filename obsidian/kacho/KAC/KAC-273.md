## Что и зачем

На detail-странице AddressPool вкладка «Адреса» (`/system/address-pools/<id>?tab=addresses`,
данные `GET /vpc/v1/addressPools/{id}/addresses`) имела разнородный набор колонок
(IP / Адрес+id-link / Зона / Project / Account / Reserved-Used). Привели к ровно
4 колонкам как в остальных списках проекта: **Имя / Идентификатор / IP-адрес / Дата создания**.

- Имя — `CopyableName` (fallback на id), обёрнут в Link на detail адреса.
- Идентификатор — `CopyableId` (моноширинный, copy-on-click).
- IP-адрес — строковое значение из payload `address.ipv4` (моноширинный).
- Дата создания — `formatDateTime(address.created_at)` (единый формат ru-RU).

Удалены дед-код запросы `/iam/v1/projects` и `/iam/v1/accounts` + `reverseLookup`
(питали только убранные Project/Account колонки) — две лишних IAM-list-выборки на
этой вкладке больше не делаются.
