# KAC-199: UI auth — sidebar login button broken + missing RequireAuth guard

**Status**: done (PR #58 merged 2026-05-25; deployed to local kacho-стенд: `docker.io/prorobotech/kacho-ui:KAC-199-1b86dcd`)
**Type**: fix (two related UI auth bugs)

## Bugs

### A — Login button broken

`ServiceSidebar.tsx::SidebarUserButton` делал `navigate("/login")`. SPA-route `/login` **не существует** — только `/auth/login`. Клик по кнопке «Войти» (LoginOutlined в нижней части sidebar) не делал визуально ничего: router не находил match → оставались на текущей странице.

### B — Anonymous users browse protected pages

`App.tsx` не оборачивал `<Layout/>`-route в `<RequireAuth/>`. Компонент существовал (`src/components/auth/RequireAuth.tsx`, KAC-127 Phase 2 fully tested) но был не подключён. Anonymous user мог открыть `/dashboard`, `/iam/*`, `/projects/.../vpc/*`, `/admin/*` — получал 401 от API и видел пустые таблицы, вместо редиректа на `/auth/login`.

## Fix

1. `ServiceSidebar.tsx`: `navigate("/login")` → `login(returnTo)` из `useAuth` (full-page assign на `kratos.loginUrl()`, тот же flow что и `<LoginButton/>`).
2. `App.tsx`: обернули `<Route element={<Layout/>}>` в `<Route element={<RequireAuth/>}>`. Вынесли `/auth/callback` и `/logout` наверх в public-блок — иначе RequireAuth создал бы infinite-loop callback↔login (Kratos session ещё не доступна на render callback'а).
3. Refactor: extract `<AppRoutes/>` как named export — для integration-тестов через `<MemoryRouter><AppRoutes/></MemoryRouter>`.
