# KAC-240 — stale-after-create: List RBAC v2 filter скрывает свежесозданный ресурс

**Status**: test (PR #121, deployed) · **Type**: fix · **Repos**: kacho-vpc

## Что и зачем

После создания VPC-ресурса он не виден в списке UI (read-your-write inconsistency).
Эмпирически подтверждено (read-only диагностика): List use-cases фильтруют строки
БД по per-object FGA allow-set (`authz.ListAllowedIDs(subject,"vpc_<res>","view")` →
`FilterByAllowedIDs`); per-object tuple пишется `fgawrite.Emit` ПОСЛЕ commit,
best-effort, а Operation done выставляется ДО записи tuple. + corelib allow-set
cache 5s. Первый refetch промахивает tuple; при silent-fail best-effort write
ресурс невидим надолго. UI корректен (refetchType:all + 3s poll + shape совпал).

## Решение (вариант 1)

List фильтровать по доступу к ПРОЕКТУ, не по per-object tuple. `repo.List` уже
folder/project-bounded → если subject может view проект, вернуть все строки. Убирает
зависимость от per-object tuple в момент List.

## Regression-фикс (2026-06-01) — list-filter дал 503 на ВСЕХ List

> [!warning] Симптом: `GET /vpc/v1/addresses` → 503 `list-filter unavailable: rpc error: code = Unimplemented desc = unknown service kacho.cloud.iam.v1.InternalIAMService`

Первый cut KAC-240 (commit 3763403) подключил project-level Check на **неверный**
conn: `listauthz.New(check.NewIAMCheckClient(iamConn))`, где `iamConn` = `extapi.iam`
= kacho-iam **public** :9090 (ProjectService). Но `InternalIAMService.Check` живёт
ТОЛЬКО на kacho-iam **internal** :9091 (`registerInternalServices`). Комментарий в
коде врал, что переиспользуется conn per-RPC gate'а — gate на самом деле строит
**отдельный** `authzConn` из `cfg.AuthZ.IAMEndpoint` (:9091).

**Fix** (`cmd/vpc/main.go`): `authzConn` (cfg.AuthZ.IAMEndpoint → :9091) строится
выше блока list-authz; list-authz теперь использует тот же `authzConn`, не `iamConn`.
Выделен seam `newListAuthz(enabled, internalIAMConn)` + RED→GREEN тест
`cmd/vpc/main_test.go` (internal-listener conn → ok; public-listener conn →
воспроизводит "unknown service"; enabled+nil → fail-fast). Startup-лог:
`list authorization ... iam_endpoint: kacho-iam-internal:9091`; live 503 устранён.

Осиротевшее наследие KAC-127 ListObjects (`internal/clients/iam_listobjects_client.go`,
dead `ListFilter.{AuthorizeEndpoint,AuthorizeTLS,TimeoutMs,CacheTTL,MaxEntries,
MaxResults,ModelID}` + chart-поля) — код не читает; рекомендован отдельный cleanup-PR
(помечено комментами в `deploy/templates/configmap.yaml` / `deploy/values.yaml`).
