# KAC-244 — gRPC keepalive отсутствует на части inter-service dial → authz Check зависает ~30с

**Status**: in-progress · **Type**: fix · **Repos**: kacho-compute, kacho-iam, kacho-vpc (sdk)

## Что и зачем

Пользователь: «каждая операция тупит». `/compute/v1/zones` и `/operations/iop…` (iam) висят
16–30с (потом 200, либо 403 `DeadlineExceeded` на 2с authz-дедлайне). vpc при этом 2–5мс.

**Диагноз (всё исключено):** pg-iam/pg-vpc idle, OpenFGA ≤28мс, iam-internal Check 10мс,
CPU `nr_throttled=0`, 0 OOM, conntrack 958/262144 0 drops, fga_outbox backlog 0. Все компоненты
здоровы по отдельности и в простое; зависания — только во всплески записи после простоя, на том
бэкенде, который активно используется.

**Корень:** соединения между сервисами простаивают между всплесками → half-open. Без keepalive
клиент не замечает, пока не придёт RPC → первый запрос всплеска висит ~30с на переустановке
(→200), либо упирается в 2с дедлайн (→403). **Асимметрия = доказательство:**
- ✅ keepalive ЕСТЬ → vpc/nlb/api-gateway **никогда не тупят**.
- ❌ keepalive НЕТ → `compute/cmd/compute/main.go:305 dialPeer` (строит `authzConn`→iam-internal:9091,
  строка 152) → compute/zones завис 30с; `iam/cmd/kacho-iam/subject_change_wiring.go:83`;
  `vpc/pkg/sdk/vpc/client.go` (внешний SDK).

## Эталон (vpc/internal/clients/builder.go)

```go
WithKeepAlive(keepalive.ClientParameters{
    Time:                opts.KeepAliveTime,
    Timeout:             opts.KeepAliveTime / 3,
    PermitWithoutStream: false,  // для idle authz-conn рассмотреть true
})
```

## Scope

1. **compute dialPeer** — добавить keepalive (первичное, доказано).
2. **iam subject_change_wiring.go:83** — то же.
3. **vpc/pkg/sdk/vpc/client.go** — то же (консистентность).
4. Исследовать **iam→OpenFGA** (SDK HTTP :8080) — возможный источник iam-operations-столла.
5. Hardening — централизовать keepalive (corelib helper) против регресса.

## Реализация (TDD, 2026-06-02)

**corelib (фундамент):**
- `grpcclient/keepalive.go` — новый пакет: `KeepaliveParams(permitWithoutStream)` /
  `KeepaliveDialOption(...)`, `DefaultKeepaliveTime=10s`, `DefaultKeepaliveTimeout=Time/3`
  (FD-1/FD-2/FD-4). Unit `keepalive_test.go`.
- `grpcsrv/server.go` — **критично (FD-2)**: `NewServer` теперь prepend'ит
  `KeepaliveEnforcementPolicy(DefaultKeepaliveEnforcement())` (MinTime=5s,
  PermitWithoutStream=true). Без этого idle-клиент с PermitWithoutStream получил бы
  GOAWAY too_many_pings от дефолтной серверной policy (MinTime=5m). `iam-internal :9091`
  получает фикс автоматически (строится через `grpcsrv.NewServer`).
- RED→GREEN: policy-level `TestDefaultEnforcement_FixesGrpcDefault` (revert фикса → RED),
  behavioral `TestNewServer_AcceptsIdleKeepalive` / `_IdleStreamSurvives` (idle-conn 16s
  остаётся READY, RPC после простоя мгновенный, стрим не оборван).
  **Finding (gRPC v1.80):** server-side GOAWAY too_many_pings от bare `grpc.NewServer`
  не воспроизводится детерминированно в unit (gRPC-go client стал консервативен в отправке
  idle-пингов) → RED-control сделан на policy-уровне (детерминирован), behavioral — GREEN-доказ.

**Consumers (seam-функции для тестируемости — grpc.NewClient не отдаёт опции назад):**
- compute `cmd/compute/main.go`: `dialPeer(addr, useTLS, idle)` + `peerDialOpts`/`peerKeepalive`.
  authz-conn → iam-internal = **idle=true**; iam(public)/vpc/vpc-internal = idle=false.
- iam `cmd/kacho-iam/subject_change_wiring.go`: `gatewayDialOpts`/`gatewayKeepalive`,
  subject-drainer conn → api-gateway-internal = **idle=true** (PermitWithoutStream).
- vpc `pkg/sdk/vpc/client.go`: `sdkDialOpts`/`sdkKeepalive` — keepalive prepend (idle=false),
  caller-opts уважаются (last-wins), insecure-дефолт только при пустых opts.

**api-gateway internal listener — verify-only:** `cmd/api-gateway/internal_grpc_listener.go`
уже permissive (`EnforcementPolicy{MinTime:10ns, PermitWithoutStream:true}`) → idle-пинги
drainer'а принимает, GOAWAY нет. **Правка не нужна.**

**KA-OF (iam→OpenFGA) — исход (c): не источник, фикс не требуется.** OpenFGA — HTTP
(`http.DefaultClient`, `openfga_client.go`), не gRPC. Дефолтный `http.DefaultTransport`:
TCP-keepalive 30s + IdleConnTimeout 90s; КАЖДЫЙ запрос обёрнут в per-op deadline
(Check 200ms / Write 1000ms, `openfga_check.go:36` / `openfga_write.go:72`) → не может виснуть
~30с. Источник iam-столла — только subject-drainer conn (KA-02), чинится этим тикетом.
Записано в acceptance-доке (KA-OF §c). НЕТ TODO (запрет #11).
