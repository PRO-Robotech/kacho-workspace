# KAC-176: newman-e2e kacho-iam authz suite regression

**Status**: to-do
**Type**: bug (regression — unblocked by KAC-174 DSN fix)

## Что и зачем

После fix'а [[KAC-174]] (DSN env-expand) kacho-nlb стэк успешно поднимается на CI. Newman-e2e теперь добегает до запуска test cases — но **kacho-iam authz suites валятся** на assertion-уровне:

* `INV sees account-A (200) — cache warm` — AssertionError
* `[W1.3-NM-01] gateway 503 Unavailable (fail-closed)` — AssertionError
* `[W1.3-NM-01] grpc code 14 (Unavailable)` — AssertionError
* `[AUTHZ-SA-NET-LS-{A2,B1}-EMPTY]`, `[AUTHZ-SANG-NET-LS-A1-EMPTY]`, `[AUTHZ-APITOK-NET-LS-B1-EMPTY]` — EMPTY status 200 fail
* Multiple `poll status 200`, `IAM Operation envelope returned` — failed assertions

**FAIL: suites with failures**: `authz-deny`, `authz-sa-apitoken`, `iam-account`, `iam-*`, `w1-nm-closeout` — `assert authz suites green` exit 1.

## Scope

* **НЕ** связано с KAC-174 (DSN fix). Регрессия preexisted — kacho-deploy newman-e2e падал на тех же step'ах с тем же exit 1 ещё до KAC-174.
* **НЕ** связано с kacho-nlb cases (320 cases пакета nlb — отдельный suite, до них не доходит).
* Скорее всего регрессия в kacho-iam (FGA writer / cache invalidate / fixture seed) или kacho-vpc (KAC-127 Phase 4 list-filter) или kacho-api-gateway (authz interceptor / Check call).

## Возможные точки исследования

1. api-gateway возвращает 503 — почему? Проверить authz interceptor + Check'и в IAM (timeout? readiness?).
2. `[AUTHZ-SA-*-EMPTY]` expects 200 EMPTY — но видимо приходит другой status или body.
3. `INV sees account-A (200) — cache warm` — проверить project-cache в kacho-vpc и IAM list-filter ([[KAC-127]] Phase 4).
4. Сравнить с предыдущим successful baseline (last green newman-e2e на kacho-deploy main).
