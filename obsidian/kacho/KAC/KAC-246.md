## Концепция «Layered Calm» (утверждена 2026-06-03)

Эстетика: премиум-дарк Linear/Vercel. Принцип: иерархия из СЛОЁВ поверхностей
(page→surface→elevated) + ритма + одного сдержанного акцента, не из границ.

**Решения заказчика:**
- Детальная страница — **левый рейл** (рефактор: поверхность/пропорции, не сливается и не «плавает»), НЕ горизонтальные табы.
- Сайдбар — **расширенный с подписями** (~220px, сгруппирован, сворачивается в icon-rail).
- Флагман-фичи в скоупе: **skeleton-загрузка, ⌘K командная палитра, брендовые empty-states, breadcrumb-контекст**.

**Фазы реализации:**
- 2A — навигационный шелл: расширенный сайдбар + top-bar breadcrumb + content page-surface + рефактор detail-рейла.
- 2B — премиум-состояния: skeleton + empty-states + ⌘K.

### Gotcha (записать — неочевидно, диагностировано Playwright'ом)

**`<table width:100%>` внутри AntD `Form.Item` с `wrapperCol={{flex:"auto"}}` выталкивает
ряд в wrap.** AntD при `flex:"auto"` ставит контролу `flex:1 1 auto` (basis=content); у
`<table>` большой min-content → label(200px)+control не влезают → `.ant-form-item-row`
(`flex-flow:row wrap`) переносит контрол на свою строку во всю ширину карточки (770px
вместо 570px у инпутов). `min-width:0` НЕ спасает. Фикс: виджет на **div-grid**
(`grid-template-columns: minmax(0,1fr) minmax(0,1fr) 40px`, `min-width:0`) — min-content
0, сжимается с колонкой, ряд не переносится. Замер: инпут и виджет L=233/R=803/W=570
идентичны. Касается любого широкого виджета в форме (CIDR-карточки, SG-rules — там Card,
ведут себя иначе, но принцип тот же).

### Gotcha: счётчик-Tag в заголовке распирает строку → «прыжок» текста

ContextBadge title — строка 16px·1.25=20px. Если в title inline-`Tag` выше
строки (height 24px) — line-box строки растёт до 24 → при `align-items:center`
текстовый блок бейджа смещается на ~2px относ. варианта без тега (detail). Фикс:
Tag.height ≤ строки (18) + span `height:20`. Любой инлайн-элемент в title держать
≤ line-height.

### Gotcha: invalidate-ключ операций = широкий prefix ресурса

Detail-страница `ResourceShell` держит данные под ключом
`[spec.id, "shell-detail", uid]`. Любой мутирующий виджет (RoutesPanel,
SgRulesPanel, CidrSection) после завершения Operation ДОЛЖЕН инвалидировать
**широкий prefix** `[spec.id]` (напр. `["subnets"]`) — React Query матчит по
префиксу, и это покрывает `shell-detail` + list. Узкие ключи (`["subnets",
"detail", id]`) НЕ совпадают с `shell-detail` → detail не обновляется (баг
CIDR-реактивности, фикс в CidrSection). Образец — RoutesPanel (`["route-tables"]`).
