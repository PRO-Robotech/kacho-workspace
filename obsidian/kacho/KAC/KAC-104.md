## Что и зачем

Платформа Kachō работает без AAA: api-gateway пускает любого, `Operation.created_by` = stub `system`, иерархия `Organization → Cloud → Folder` (resource-manager) дублирует «account/project» из IaaS без своей семантики. Эпик вводит auth/authz:

- **`kacho-iam`** (новый сервис, REBAC control-plane): Account / Project / User / ServiceAccount / Group / Role / AccessBinding.
- **Zitadel** (external OIDC): identity, signup, JWT, JWKS.
- **OpenFGA** (external REBAC, Zanzibar): tuple-store, `Check`-API.
- **api-gateway** получает auth-interceptor (JWT→Principal через `kacho-iam.SubjectService.Lookup`) и authz-interceptor (`openfga.Check`).

`Account` заменяет Organization+Cloud, `Project` заменяет Folder. Principal пробрасывается gRPC-metadata в backend, оседает в `operations.principal_*`. UI получает блок IAM. `kacho-resource-manager` выключается (E5).

Acceptance: `docs/specs/sub-phase-2.0-iam-overview-acceptance.md` (882 строки, DRAFT).
