## Fixes

### F2 — `IAM-USR-INV-CRUD-OK` invite missing `projectId` (24 asserts)

`tests/newman/cases/iam-user.py` — invite body теперь шлёт `projectId: {{projectA1Id}}`
parity со server contract (`user_service.proto:117-133` + `invite.go:118-123` требует
`project_id` paired с `role_id`). Без него сервер возвращал 400 «project_id: required
when role_id is set» → каскадно 23 poll-op 404 на stale `opId`.

### F3 — JIT-pending seed rows never INSERT'd (33 asserts)

`tests/newman/scripts/run.sh::reseed_jit()` — расширена: теперь INSERT 3 seed rows
(`jp_00000kac132seed01/02/03`) + 3 parent eligibility rows
(`jite_kac182_seed_a/b/c`) идемпотентно (ON CONFLICT DO NOTHING), затем RESET
decision-state — как раньше. Users (AAA/INV) резолвятся через `external_id` lookup.

Закрыт «TODO KAC-127-seed never closed» pattern (§11 violation в исходной форме).
Docstring + pre_script TODOs в `iam-jit-pending.py` обновлены — указывают на
reseed_jit как источник истины seed-данных.

### F6 — `inv-get-account-allow-warm-cache` 8-poll retry (1 assert)

`tests/newman/cases/authz-deny.py::inv-get-account-allow-warm-cache` — обёрнут
в 8-poll retry (~1.6s @ 200ms), симметрично post-revoke loop в той же кейсе
(lines 688-706). FGA tuple-write async via W1.5 fga_outbox drainer — Operation
done ≠ tuple visible.
