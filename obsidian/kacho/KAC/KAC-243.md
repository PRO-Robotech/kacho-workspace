# KAC-243 — SG↔Network: обязательна + immutable; правила SG→SG только в одной сети

**Status**: in-progress (acceptance APPROVED, код не начат) · **Type**: feature · **Repos**: kacho-proto, kacho-vpc, kacho-ui · **Parent**: [[KAC-239]]

## Что и зачем

Из [[KAC-242]] (actions в шапке) пользователь уточнил продуктовое требование. Направление
**развернулось дважды**: сначала «привязать network-less SG» (отвергнуто — network-less SG не
должно существовать), затем — финальное:

- **SG.network_id ОБЯЗАТЕЛЬНО** при создании (откат недавнего «опционально», kacho-proto#8).
- **network_id IMMUTABLE** — переключить SG на другую сеть нельзя (уже create-only на бэке; закрепляем + Move-guard).
- **Правила SG→SG только в одной Network**: правило с target=другая SG валидно только если та SG в той же сети
  (SG в разных сетях физически не видят друг друга → cross-network SG-rule бессмысленно).

## Acceptance (APPROVED)

`docs/specs/sub-phase-securitygroup-network-mandatory-and-same-network-rules-acceptance.md` —
19 сценариев (SG-NET-01…19, группы A–G), 7 fixed-decisions, APPROVED acceptance-reviewer'ом.
Ключевые решения: cross-network/non-existent target → `INVALID_ARGUMENT`; текст `"network_id required"`;
same-network — service-layer валидация (network_id immutable → не TOCTOU; нормализация target-table = redesign-эпик);
sync fast-fail + async backstop; **Move cross-project network-bound SG → FAILED_PRECONDITION** (SG-NET-19);
миграция `SET NOT NULL` новым файлом; `buf breaking` зелёный (custom (required) option).

## План работ (топологически)

1. **kacho-proto**: `(required)` на `CreateSecurityGroupRequest.network_id`; regen gen/; buf lint/breaking.
2. **kacho-vpc**: Create требует network_id; новая миграция `network_id SET NOT NULL`; same-network rule
   validation (Create rule_specs + UpdateRules + UpdateRule — инжект SG/network reader в use-cases);
   Move-guard; Update оставляет network_id вне mask (тест). Integration (testcontainers) + newman.
3. **kacho-ui**: SG-create network_id required (preset+lock на network-табе); rule SG-picker same-network фильтр;
   убрать любой привязать-stub.
