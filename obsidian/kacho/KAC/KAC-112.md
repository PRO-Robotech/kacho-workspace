## Что и зачем

E0 PR-3 (основной `kacho-iam`) поднял **только Account** ресурс end-to-end (handler + use-cases + CQRS repo + sentinel mapping + integration tests + newman). Остальные 6 ресурсов из доменной модели E0 — Project / User / ServiceAccount / Group(+GroupMember) / Role / AccessBinding — на момент E0 merge'а присутствуют **только** в:
- `internal/domain/*.go` — newtypes + Validate (по skill `evgeniy` §4)
- `internal/repo/kacho/*/iface.go` — Reader/Writer port-интерфейсы
- `internal/migrations/0001_initial.sql` — DB schema (squashed baseline)
- `kacho-proto/proto/kacho/cloud/iam/v1/*.proto` — proto-stubs (генерация commit-нута)

Этот тикет — **закрыть оставшиеся 6 ресурсов** по тому же pattern'у, что Account:

1. `apps/kacho/api/<resource>/` — use-cases-per-file (create/get/list/update/delete + Move для Project, AddMember/RemoveMember/ListMembers для Group, UpsertFromIdentity для User-internal).
2. `repo/kacho/pg/<resource>.go` — pgxpool реализация Reader/Writer.
3. `repo/kacho/repomock/<resource>.go` — mock для unit-тестов use-case'ов.
4. `repo/kacho/pg/maperr.go` — расширить constraint-name → gRPC mapping.
5. Integration тесты (testcontainers) per ресурс — особенно **concurrent-race-сценарии** для Project `Move` (CAS) и AccessBinding `Create` идемпотентность (UNIQUE).
6. Newman cases — ≥1 happy + ≥1 negative per ресурс (минимум 10 cases-файлов; см. acceptance §10).
7. Handler регистрация в `cmd/kacho-iam/main.go` + api-gateway routing prefix-аккумуляция (`prj`, `usr`, `sva`, `grp`, `rol`, `acb` + `iop`).
