# KAC-191: kacho-iam — permission_catalog.json mirror has empty `permission` fields

**Status**: test
**Type**: fix
**Repos**: kacho-iam
**PRs**: PRO-Robotech/kacho-iam#49 (open)
**YT**: https://prorobotech.youtrack.cloud/issue/KAC-191

## Что и зачем

`internal/apps/kacho/api/internal_iam/embedded/permission_catalog.json` (embedded в iam binary через go:embed, используется `InternalIAMService.ListPermissions`) содержал 300+ entries, но во всех `"permission": ""`.

`list_permissions.go::loadCatalogPermissions` фильтрует empty (`if e.Permission == "" continue`) → runtime список пустой → `ListPermissions` возвращает `{}`.

Newman case `IAM-INT-OK-INT-LISTPERMS` асёртит non-empty array → FAIL.

## Fix

1. Copy `kacho-proto/gen/permission_catalog.json` → embedded location (300 populated entries)
2. Add Makefile target `sync-permission-catalog`
3. RED-first integration test `TestLoadCatalogPermissions_NonEmpty` — asserts ≥100 entries + format spot-check

## RED → GREEN

- OLD catalog: `FAIL: permission catalog has 0 entries, want >= 100`
- NEW catalog: `ok  ... 0.005s`
