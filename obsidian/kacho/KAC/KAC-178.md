# KAC-178: Stand prod-readiness — 5 backend gaps

**Status**: in-progress
**Type**: epic (5 sub-gaps, см. design-doc)

## Что и зачем

После [[KAC-171]] (UI NLB integration) + [[KAC-175]] (port-portable Kratos/Hydra) UI полностью починен. Однако downstream **backend endpoints** падают на 5 разных уровнях AuthZ / catalog / proto:

- **§1** [vpc#112] list-filter PermissionDenied → Unavailable (code 14 leak) + FGA relation mismatch
- **§2** [vpc#114] W1.4 principal propagation — Operation.created_by=anonymous
- **§3** [compute#29] Compute catalog gap + FGA model cluster wildcard
- **§4** [api-gw#33] NLB proto-annotations + ListenerService catalog
- **§5** Catalog source-of-truth clarification (kacho-iam vs kacho-api-gateway)

Дизайн-doc: `docs/specs/2026-05-24-stand-prod-readiness-design.md`.

## Discovery (2026-05-24)

Большая часть work уже сделана в открытых branch'ах:
- **§2** W1.4: `kacho-corelib/auth/propagate.go` done; `kacho-vpc/cmd/vpc/main.go` mounts `UnaryPrincipalExtract`; `kacho-vpc/.../check_client.go` + `kacho-compute/.../check_client.go` оба обёрнуты `auth.PropagateOutgoing`. **Не хватает**: `kacho-compute/cmd/compute/main.go` НЕ mountит `UnaryPrincipalExtract`.
- **§3** + **§4 catalog**: `kacho-api-gateway` branch `KAC-zones-catalog` — populated compute Region/Zone Get/List + ListenerService entries + ComplianceReportService + JitPendingService. Не PR'нут.
- **§1 wrap fix**: НЕ сделан — 8 use-case files в `kacho-vpc/internal/apps/kacho/api/<resource>/list.go` оборачивают `lerr` в `Unavailable("list-filter unavailable: ...")` безусловно. Нужно: если `lerr` это PermissionDenied — возвращать как-есть.
- **§1 Helm override**: НЕ сделан. Branch `kacho-deploy KAC-179-dev-list-filter` пустая.
- **§4 UI path mismatch**: UI hits `/nlb/v1/loadBalancers`, но proto annotation `/nlb/v1/networkLoadBalancers` (verbatim YC). Нужно: fix UI к canonical paths.
- **§3 FGA cluster wildcard**: NotImplemented. Catalog entries для Region/Zone используют `scope_extractor.object_type=cluster, from_request_field='*'` → нужен FGA tuple bootstrap для `cluster:cluster_kacho_root` или schema rule `cluster_viewer`.
