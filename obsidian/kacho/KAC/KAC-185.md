# KAC-185: F4 — Internal IAM RPCs lack google.api.http annotation

**Status**: test
**Type**: fix
**Repos**: kacho-proto, kacho-api-gateway
**PRs**: PRO-Robotech/kacho-proto#24, PRO-Robotech/kacho-api-gateway#38
**YT**: https://prorobotech.youtrack.cloud/issue/KAC-185

## Что и зачем

10 newman failures (suite `iam-internal-only-check`, 10/23 asserts) on CI run
`26358993224`: every `IAM-INT-OK-INT-…-ON-INTERNAL-OK` test returned 403
`catalog: no entry for method; unauthenticated`.

Cause: 4 Internal IAM RPCs (`UpsertFromIdentity`, `LookupSubject`,
`ListPermissions`, `Check`) had no `option (google.api.http)`. grpc-gateway
created no REST routes → api-gateway authz middleware's REST→FQN resolver
fell back to path-literal → permission catalog (which DOES contain the FQNs
as `<exempt>`) was never reached → blanket deny.

Fix:
- **kacho-proto KAC-185**: add `google.api.http` to 4 RPCs in
  `internal_user_service.proto` + `internal_iam_service.proto`. Buf lint +
  breaking clean; gen/ regenerated; `permission_catalog.json` byte-identical.
- **kacho-api-gateway KAC-185**: 4 new entries in `rest_route_table_gen.go`;
  register `InternalIAMService` in `restmux/mux.go`; add 4 FQNs to
  `authz_public_allowlist.go` (catalog `<exempt>` still enforces auth, but
  cluster-internal callers carry no JWT). Ref-pin proto to KAC-185 in CI.

§«Запреты» #6 isolation preserved: enforced at LISTENER layer (TLS endpoint
firewalled; gRPC director's `HasInternalSuffix` blocks Internal* on public).
