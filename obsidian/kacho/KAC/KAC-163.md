# KAC-163: W1.5 — Remediation Chunk 1

**Status**: in-progress (PR awaiting CI/merge) · **Parent**: [[KAC-136]] · **Epic**: [[KAC-134]]
**Acceptance**: `docs/specs/sub-phase-W1.5-remediation-chunk1-fga-grant-write-acceptance.md` ✅ APPROVED

## Closes 7 findings

- **#8** AB.Delete atomic revoke (was sync `DeleteTuples`, no atomicity guarantee)
- **#16** AB.Create atomic emit (was sync `WriteTuples`, split-brain DB/FGA on FGA outage)
- **#47/#48** `PermissionsToRelations` (custom roles no longer collapse to single relation by name regex — derives from spec.permissions)
- **#50** JIT auto-grant FGA emit (path silently bypassed FGA write)
- **#51** JIT Approve/Expiry grant/revoke (was wrong `EmitSubjectErasure` — CAEP-deletion! — replaced with proper FGA grant/revoke)
- **#52** BG.ApproveB `cluster_admin_grants` + FGA emit (mirrors bootstrap_admin pattern)

## Implementation

- Commit `4301cb7` on `KAC-163` branch (kacho-iam)
- Migration 0024 — UNIQUE on `cluster_admin_grants(subject_user_id, project_id)` per §запрет #10 (DB-level invariant, not software-side)
- New `internal/repo/kacho/pg/fga_outbox/` package — `EmitWriteTx` / `EmitDeleteTx`, parity with W1.2 emit pattern (subject_change_outbox)
- New `internal/authzmap/permissions_to_relations.go` — supersedes both `tuples.go::roleNameToRelation` and `authzmap.RoleNameToRelation` (name-regex based); now derives FGA relations from `role.spec.permissions[]`
- ALL grant/revoke paths routed via `fga_outbox` in same writer-tx:
  - `AccessBindingService.Create` / `Delete`
  - JIT auto-grant (`activate`)
  - JIT pending `Approve` / `Expiry` (was wrong subject-erasure call)
  - `BreakGlassService.ApproveB`
- Reuses W1.1 drainer + W1.2 emit pattern (LISTEN/NOTIFY → FGAApplier)
