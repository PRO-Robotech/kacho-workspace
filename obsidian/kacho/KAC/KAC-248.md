## Что и зачем

Отдельный публичный статический docs-сайт `docs.kacho.local` на **Astro Starlight**,
визуально единый с kacho-ui («Layered Calm»), контент в MD/MDX (self-maintained).
Принцип **«один источник — три читателя»**: единый корпус обслуживает пользователя-
оператора, разработчика публичного API и AI-агентов одним и тем же контентом по
построению. API-reference **генерируется из `kacho-proto`** (proto→OpenAPI),
фильтруется до публичной поверхности, не расходится с API.

Дизайн-документ (источник истины):
`docs/superpowers/specs/2026-06-05-kacho-docs-2026-design.md` (research → синтез →
adversarial-критик → ревизия).

## Ключевые решения

- **Infra-leak guard (критично):** allowlist публичной поверхности **переезжает в
  `kacho-proto`** как канонический источник (split-brain исключён по построению),
  api-gateway импортирует. `cmd/openapi-filter` роняет Internal*/AddressPool.
  Проверено: 6 `Internal*`-proto несут `google.api.http` → http-наличие НЕ
  дискриминатор public/internal, allowlist обязателен.
- Тема: маппинг `--kc-*` → `--sl-color-*` в одном CSS (токены верифицированы).
- MVP: P1 read-only reference · P2 persona-tabs · P3 llms.txt+copy-as-md ·
  P4 static OperationEnvelope · P6 compiled-snippets · P7 Pagefind ⌘K+тема.
  Phase 2: Try-it (Scalar+Prism), SDK-сниппеты, live OperationRunner, Hurl-verified,
  docs-MCP. Phase 3: Ask-AI RAG, topology, semantic, версии.

## 🎯 Запуск (2026-06-05)

**docs.kacho.local поднят на kind-стенде `kacho`.** Astro Starlight static → nginx
(образ `kacho-docs:dev`, helm release `kacho-docs`, pod Running 1/1, ingress `docs`).

**Доступ:** `http://localhost:28080/` (kind host-mapping `0.0.0.0:28080→80`) или
node `172.18.0.2:80`, заголовок `Host: docs.kacho.local`. Smoke зелёный: home /
quickstart / 8 концепций / how-to / api-guide / `llms.txt` / per-page `.md` отдаются;
404 (directory-index, не SPA); Cache-Control HTML=`no-store`, assets=`immutable`.

**Что затронуто вне kacho-docs (флаг для пользователя):**
- `kacho-deploy` НЕ редактировался — деплой standalone helm-релизом (`helm install
  kacho-docs project/kacho-docs/deploy`), не вшит в umbrella `dev-up`. Опционально:
  добавить `build-docs` target + docs-subchart для переживания `dev-up/down`.
- admission-webhook стенда `kacho-umbrella-ingress-nginx-admission` был **сломан до меня**
  (x509 unknown authority — блокировал ВСЕ новые ingress'ы); пропатчен
  `failurePolicy→Ignore` (реверсивно; чинится на следующем `helm upgrade` umbrella).
- `kacho-docs` — **локальный репо без GitHub-remote** (публичный репо не создавал —
  outward-действие, по запросу).
