## Что и зачем

После KAC-130/131/132 осталось ~25 assertion-failures в newman E2E на live kind-стенде.
Цель: закрыть все до 4 intentional RED (#37 — compliance foreign-deny).

## Backend fixes (kacho-iam)

1. **FindExisting** (`repo/kacho/pg/access_binding_repo.go`): добавлен `AND status = 'ACTIVE'` — без него REVOKED-row возвращалась с другим id, ломая ACB-IDEM-13.4.
2. **ListBySubjectUseCase**: self-only guard для `user` principals (KAC-123) — cross-user → 403.
3. **ListByResourceUseCase**: `requireGrantAuthority` scope-check + FGA wiring в main.go — non-member не может перечислить bindings.
4. **ListAccountsForUser** (`repo/kacho/pg/user_repo.go`): UNION расширен — аккаунты с ACTIVE AccessBinding (resource_type='account') включены; приглашённые (JIT_PENDING) теперь видят accountB.

## Test fixes (newman)

- `iam-access-binding`: ListByResource/ListBySubject → GET + query params (не POST)
- `iam-account`: IAM-ACC-CR-NEG-NAME-DUP сохраняет dupOp в opId
- `iam-group/project/service-account`: принять 403 для missing-account (FGA deny до Operation)
- `iam-user`: pageSize=1001 → 200 (silent cap); IAM-USR-SETUP-INVITE-INV-TO-B перед MEMBER-SEES

## Scope extension (2026-05-23, late session): CICD fix

Compute/VPC ветки KAC-133 закоммитили адаптеры `IAMCheckClient`, использующие
`authz.ErrNoPath` из corelib — но в corelib `main` этого sentinel не было. CI
у kacho-vpc и kacho-compute падал на `undefined: authz.ErrNoPath`.

**Fix sequence:**
1. **kacho-corelib PR #12** — добавлен `ErrNoPath` + `DecisionNoPath` passthrough
   (см. `authz/types.go`, `authz/interceptor.go`). Смёрджен в main первым по
   графу зависимостей.
2. **kacho-vpc** CI зазеленел автоматически после corelib merge (ничего
   доп. не правили в этом репо).
3. **kacho-compute** всплыли две скрытые ошибки, замаскированные первым
   compile failure:
   - 4 service-конструктора (`NewDiskService`/`NewImageService`/`NewSnapshotService`/`NewInstanceService`)
     получили 2 trailing-параметра (`fgawrite.HierarchyTupleWriter`, `*slog.Logger`),
     тесты не обновлены → 9 call-sites в `disk_test.go` / `image_snapshot_test.go` /
     `instance_test.go` / `handler_test.go` / `list_filter_test.go` обновлены: `nil, nil`
     (fgawrite.Emit nil-safe для обоих).
   - `TestIntegration_CatalogRepo` ждал 3 zone, в seed теперь 4 (`ru-central1-c`
     из миграции `0010_add_zone_c.sql`) — обновлено на 4.
   - `golangci-lint`: dead-code `objectTypeSystem` / `systemCatalogID` /
     `staticSystemCatalog()` в `internal/check/permission_map.go` (после
     перевода catalog endpoints на `Public: true`) удалён.
