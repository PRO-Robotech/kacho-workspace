## Что и зачем

gRPC/HTTP convention (RFC 7235, gRPC status code guide):
- **Missing/invalid credentials** → `UNAUTHENTICATED` (16) → HTTP 401
- **Authenticated subject, access denied** → `PERMISSION_DENIED` (7) → HTTP 403

The authz interceptor (`kacho-api-gateway/internal/middleware/authz.go`) returned
`outcomeDeny` → `buildGRPCDenyStatus()` → code 7 / HTTP 403 for BOTH paths.
When a request had no JWT at all (no authenticated subject), it should return 16/401.

**Root cause (file:line)**:
- `authz.go:472-483` (subject extraction) — `outcomeDeny` for `!ok` → now `outcomeUnauthenticated`
- `authz.go:434-444` (exempt entry gate) — same pattern

## Fix

- Added `outcomeUnauthenticated` to `decisionOutcome` enum.
- `decide()`: both missing-subject paths return `outcomeUnauthenticated` instead of `outcomeDeny`.
- `decision.gRPCStatus()` dispatches to `buildGRPCUnauthStatus()` (code 16) for unauthenticated.
- Unary/Stream/HTTP handlers: handle `outcomeUnauthenticated` — gRPC returns code 16 error,
  HTTP calls new `writeHTTPUnauth()` (401 + `code:16` JSON + `WWW-Authenticate: Bearer`).
- `permission_denied_response.go`: added `buildGRPCUnauthStatus()` + `writeHTTPUnauth()`.
- Authenticated-deny (FGA check returned false) unchanged: 7/403.

## Newman alignment (kacho-iam)
