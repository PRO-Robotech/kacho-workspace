## Сессия 2026-05-31 — VPC detail UX + 4 бага

- **СИСТЕМНО: stale list/detail после async-операции + 400 на createOnly в update_mask** (kacho-ui `1b83601`):
  - **Root cause stale**: `invalidateResourceList` звал `invalidateQueries` с дефолтным `refetchType:"active"` → неактивные (ещё-не-примонтированные при create-then-navigate) списки лишь помечались stale, не рефетчились → видно старое. Фикс: централизованный `invalidateResourceKeys(qc, resourceId)` с `refetchType:"all"` (рефетчит и неактивные) + одна отложенная safety-инвалидация ~1.2с (окно op.done→row-visible). Покрывает list + detail + shell-detail (resourceId-first) + ref-name + networks→SG + breadcrumb/dash. Для ВСЕХ ресурсов сразу. (Заменяет точечные правки detail/shell-detail выше.)
  - **Root cause 400** `unknown field in update_mask: create_default_security_group`: `computeUpdateMask` (ResourceFormDialog.tsx) пропускал hidden/immutable/editHidden/`_`-prefixed, но НЕ `createOnly`. Поле `create_default_security_group` — `createOnly` (write-only при Create). Фикс: `computeUpdateMask` также `continue` на `createOnly`. Sweep: это ЕДИНСТВЕННОЕ createOnly-поле в registry → покрыто полностью; 4 hand-built-mask формы (NIC/SG/AddressPool/Subnet) createOnly-полей не содержат.
  - Уроки: (1) invalidateQueries по умолчанию НЕ рефетчит неактивные запросы — для create-then-navigate нужен `refetchType:"all"`; (2) update_mask должен исключать и `createOnly`, не только immutable/editHidden; (3) list-хук имеет `refetchInterval:3s` (`use-resource-list.ts`) — авто-догон есть, но refetchType:all убирает задержку до 3с.
- **Bug fix: Обзор показывал старое состояние после операции** (kacho-ui `7d1c167`): `invalidateResourceList` инвалидировал только `[resourceId, "detail"]` (ключ ResourceDetailPage/EditPage), но Обзор RouteTable (и overviewBelow, где живёт RoutesPanel) рендерит **ResourceShell** с ключом `[spec.id, "shell-detail", uid]` — он НЕ инвалидировался → Обзор не рефетчился после операции. Фикс: инвалидируем оба resourceId-first ключа — `[resourceId, "detail"]` И `[resourceId, "shell-detail"]` (+ SG side-effect аналогично). (Промежуточный 6eb4d2f с `["detail", resourceId]` был неверен — TanStack prefix-матчит с начала массива, реальные ключи resourceId-first; откатан в 7d1c167.) Урок: detail-ключи проекта — `[spec.id, "detail"|"shell-detail", uid]`, инвалидация всегда resourceId-first и должна покрывать обе формы.
- **RoutesPanel no-jump + стиль кнопки + FieldMask fix** (kacho-ui `79561ad`/`1ffe5d4`/`b1762ef`):
  - Таблица не скачет при входе в edit: `table-layout:fixed` + `<colgroup>` с явными ширинами `calc((100%-48px)/2)×2 + 48px`; колонка действий (🗑) присутствует ВСЕГДА (пустая в read) → число колонок постоянно; фиксированная `height:ROW_H(41)` + `vertical-align:middle` + borderless Input (`padding:0`, height/lineHeight под ROW_H) → высота строки одинакова для текста и инпута. Блок ограничен `maxWidth:760` (2 колонки не на весь экран).
  - Кнопка «Редактировать» — default-стиль `<Button icon>` как resource-level (ResourceShell:265), не `type=primary`.
  - **Bug fix**: PATCH RouteTable падал 400 `FieldMask.paths contains invalid path: static_routes` — FieldMask JSON-пути по googleapis-маппингу camelCase; шлём `update_mask:"staticRoutes"` (конвенция ResourceEditPage `camelMaskFromObject`). Урок: update_mask всегда camelCase, snake protojson отвергает.
- **RoutesPanel — единая таблица** (kacho-ui `7d2c3b1`, после `5cd8801`): read («Обзор») и edit — ОДНА таблица. «Редактировать» делает ячейки editable (seamless borderless Input, padding 4px 0 — без визуального прыжка) + колонка 🗑 (per-row delete) + dashed-строка «Добавить маршрут» снизу (tfoot colSpan).
