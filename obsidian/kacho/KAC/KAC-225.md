# KAC-225: gateway WhoAmI GET /iam/v1/me отсутствует в route-table → 403

**Status**: done
**Type**: fix

**YT**: https://prorobotech.youtrack.cloud/issue/KAC-225

## Что и зачем

`AuthorizeService.WhoAmI` (GET `/iam/v1/me`) есть в `permission_catalog.json` как `<exempt>`, но **отсутствовал** в `internal/middleware/rest_route_table_gen.go`. Authz-middleware резолвит REST-путь→gRPC-FQN через эту таблицу; без записи fqn получался мусорный `//iam/v1/me`, `found=false`, и exempt-bypass (`authz.go: if found && IsExempt()`) не срабатывал → **403 "catalog: no entry for method"**. Ломало весь UI permission-bootstrap (`usePermissions` завязан на `/iam/v1/me`).

Route-table не перегенерили при синке WhoAmI в catalog (commit 9dcdca1). Предсказано аудитом адаптации UI (gateway-rest lens).

## Фикс

Добавлена запись `{GET, /iam/v1/me, AuthorizeService/WhoAmI}` в `rest_route_table_gen.go` + regression-кейс в `rest_router_test.go`. Stopgap-правка generated-файла; proper — регенерация route-extractor'ом из kacho-proto. Проверено на стенде: unauth `/iam/v1/me` → 401 (было 403).
