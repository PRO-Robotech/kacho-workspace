## Что и зачем

F5 = 4 newman failures в suite `authz-sa-apitoken` (CI run 26358993224).

Корневая причина (3-fold, см. workspace `docs/specs/newman-80-failures-finding.md` v2):

1. `kacho-vpc/cmd/vpc/main.go:221-230` вызывает iam `AuthorizeService.ListObjects` как service-to-service bootstrap peer call **без** `PerRPCCredentials` (это preflight authz-query, не user-request) → приходит anonymous.
2. `kacho-iam/internal/authzguard/interceptor.go:50-53` `readonlySuffixes` использует `strings.HasSuffix` (per reviewer A1, line 44) — `"ListObjects"` / `"ListSubjects"` не оканчиваются на `"List"`, поэтому suffix-match не покрывает.
3. `whitelistFullMethod` (line 61-70) не содержал ни `ListObjects`, ни `ListSubjects`.

Default-deny → `PermissionDenied` → vpc маппил в 503 «list-filter unavailable».

Fix: явные `FullMethod` whitelist-записи для **обоих** RPC (`ListObjects` + симметричный `ListSubjects` — тот же класс peer-bootstrap, чтобы избежать future-регрессии той же причины). `ExpandRelations` оставлен out-of-scope (не задействован F5).

Defense-in-depth для production-strict cross-pod authn — план W3.3 SPIRE+Cilium (mTLS + NetworkPolicy + SPIFFE-SVID). Whitelist — immediate correctness fix, **не** TODO/follow-up (workspace §11).
