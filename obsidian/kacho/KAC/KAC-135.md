## Что и зачем

Подготовка инфры для всех Wave'ов KAC-134:

1. **coverage.py** — RPC→case-id gate (парсит `kacho-proto/proto/kacho/cloud/iam/v1/*.proto` +
   `tests/newman/collections/*.json`, мапит через `google.api.http`, exit-code 1 если
   covered% < `--min`). Без него «100% покрытие» не измерить.

2. **CI matrix** — newman-e2e.yml в обоих репо гоняет ВСЕ collection-сюиты (сейчас только
   authz-deny / authz-sa-apitoken), fail на любом failed-assertion + coverage gate.

3. **OpenFGA HA-mini bootstrap** — Helm post-install/post-upgrade Job, идемпотентно создаёт
   store `kacho-iam` + загружает AuthorizationModel; OpenFGA up to 2 replicas. Без bootstrap
   kacho-iam не может писать FGA-tuples (W1-drainer полагается на наличие store).
