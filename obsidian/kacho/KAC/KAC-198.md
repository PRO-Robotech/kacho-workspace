# KAC-198: user-menu в sidebar + service-prefix fix

**Status**: test (deployed на стенд, PR open)
**Type**: fix (UX + navigation)
**Repos**: kacho-ui

## Что и зачем

Two-in-one UI batch:

### Part 1 — user-menu в sidebar (вместо header)

Раньше `<HeaderAuth/>` (avatar + email + dropdown Профиль/Выйти) был в шапке справа.
Перенесён в нижнюю часть sidebar — компонент `SidebarUserButton`:
- залогинен → Avatar (initials) → клик → Dropdown с email + Профиль (→/iam/users) + Выйти
- !user → `<LoginOutlined/>` → /login
- loading → Spin

Dropdown placement = `topRight` (popover уходит вверх+вправо, не выходит за нижний край sidebar).

В `service-modules.tsx::COMMON_BOTTOM` статический profile leaf удалён —
рендеринг user-кнопки теперь не через NavLeaf, а отдельным компонентом.

### Part 2 — service-prefix fix в detail-links

`RefNameLink`, `NetworkDetailPage::useChildColumns`, `ResourceDetailPage::backHref`
строили URL `/projects/<pid>/<spec.route>` БЕЗ service-сегмента (vpc/compute/nlb).
SPA-router в App.tsx ожидает `/projects/:pid/vpc/<route>/:uid`,
`/projects/:pid/compute/<route>/:uid`, и т.д. → SPA fallback на blank/404.

User-repro: Network detail → таб Subnets → клик по подсети →
`/projects/<pid>/subnets/<id>` → 404.

Fix: новый helper `resourceProjectPath(specId, projectId)` в
`src/lib/resource-registry.tsx` — возвращает правильный SPA-путь с
service-сегментом по spec.id (compute-* → compute, NLB → nlb, IAM → null,
остальные → vpc). 3 callsite'а переведены.
