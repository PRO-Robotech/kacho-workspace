**Status**: in-progress

**YT**: pending

## Что и зачем

Финальный pruning-проход. После phases 1-3 от 47k → 34k LOC. Цель — спустить до **~30k**:
- **DROP JIT** (Just-In-Time access) — 3 пакета (`jit_eligibility`, `jit_pending`, `jit_user`) + связанные services/domain/migration. ~−3500 LOC. **Не востребовано** в MVP: модель Account/Project + RBAC + break-glass для emergency достаточна.
- **Simplify GDPR** — оставить cascade-delete для PII; убрать full erasure pipeline (data-discovery, anonymization, retention policies, erasure jobs). ~−2000 LOC.
- **KEEP**:
  - `break_glass_grant` — emergency admin access (MFA + audit + TTL). Core admin mechanism, требуется ([[KAC-196]] partial fix).
  - `cluster_admin_grant` ([[KAC-196]]) — bootstrap-grant of admin role to a specific user.
  - Cascade-delete user → memberships → tokens (для GDPR-baseline).
