# KAC-192: kacho-iam — w1-nm-closeout newman cases use unregistered REST paths

**Status**: open (whitelisted in KAC-188 PR #46)
**Type**: bug
**Repos**: kacho-iam
**PRs**: (TBD — KAC-192 dedicated PR not yet opened)

## Что и зачем

Multiple cases в `tests/newman/cases/w1-nm-closeout.py` используют REST пути которых нет в proto / api-gateway route table:

| Wrong path | Used by | Correct path |
|---|---|---|
| `/iam/v1/check` | W1.5-NM-01/02/05 | `/iam/v1/authorize:check` |
| `/iam/v1/jitPendingApprovals/...` | W1.6-NM-12/13, W1.5-NM-04 | `/iam/v1/jitPending/...` |
| `/iam/v1/complianceReports/{id}` | W1.6-NM-14 | OK exists, may need fix elsewhere |
| `/iam/v1/breakGlassRequests/...` | W1.5-NM-06 | not registered |
| `/iam/v1/jitEligibilities/...` | W1.6-NM-16 | needs verification |

## Impact

11 assertion failures в `w1-nm-closeout` suite когда run.sh её запускает. Whitelisted в KAC-188 PR #46 workflow assert step через step-name pattern alternation.

## Long-term fix

Update cases to correct paths (preferred) OR remove cases with documented reason. Workflow whitelist clamps to 0 if cases become GREEN organically → self-healing.
