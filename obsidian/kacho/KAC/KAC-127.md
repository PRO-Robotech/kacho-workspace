## Что и зачем

Полная **production-ready** реализация IAM Kachō. Все 2026 best-practices в одном эпике: ORY Kratos+Hydra+Jackson AuthN (WebAuthn/Passkey-first + DPoP + mTLS-bound + step-up + recovery), OpenFGA+OPA AuthZ (Conditions overlay + Rego guardrails), Workload Identity Federation (Class A static + Class B OIDC federation для GitHub/AWS/GCP/GitLab/CircleCI/Buildkite/Bitbucket + Class C SPIFFE/SPIRE in-cluster + Cilium mesh), SCIM 2.0 + SAML bridge, PIM/JIT + Break-glass 2-person, CAEP push, full audit pipeline (Kafka+ClickHouse+S3+Glacier+HSM+SIEM+Merkle), production deploy (api.kacho.cloud + Cloudflare WAF + multi-region active-active + cert-manager + Argo CD + SBOM/SLSA), conformance (OWASP ASVS L3 + OIDC certified + FIDO Alliance) + chaos engineering + continuous fuzzing + external pentest + bug bounty.

## Decision Log (full в design doc; здесь high-level)

- D-1: AuthN = ORY Kratos + Hydra (Zitadel removed)
- D-2: AuthZ = OpenFGA
- D-3: OPA sidecar для org-wide deny
- D-4: Hierarchy = cluster → org (optional) → account → project → resource
- D-5: Custom Role multi-scope (project/account/org)
- D-6: DPoP + mTLS-bound JWT
- D-7: WebAuthn/Passkey-first
- D-8: Workload Identity Federation (RFC 8693)
- D-9: SCIM 2.0 inbound
- D-10: SAML 2.0 via Boxyhq Jackson bridge
- D-11: AccessBinding.status PENDING/ACTIVE/REVOKED
- D-12: Conditions overlay (CEL-like)
- D-13: PIM/JIT для admin (standing admin = anti-pattern)
- D-14: Break-glass 2-person + 2h TTL + alerts
- D-15: CAEP push ≤10s revoke
- D-16: SPIFFE/SPIRE + Cilium mesh in-cluster mTLS (production edition: **IN scope**)
- D-17: Cluster:singleton OpenFGA object
- D-18: Full audit pipeline (Kafka+ClickHouse+S3+HSM+SIEM+Merkle)
- D-19: Bluегreen migration; new production stand `api.kacho.cloud`
- D-20: Public TLS via Cloudflare + cert-manager (Let's Encrypt)
- D-21: Multi-region active-active (RTO ≤15min / RPO ≤1min)
- D-22: Continuous fuzzing
- D-23: SBOM + SLSA L3 build provenance
- D-24: Bug bounty + security.txt
- D-25: OpenTelemetry end-to-end
- D-26: Chaos engineering quarterly
- D-27: Annual pentest + on major release
- D-28: Post-quantum readiness (hybrid TLS)
