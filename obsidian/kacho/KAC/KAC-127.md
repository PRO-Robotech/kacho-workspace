## newman-e2e green-up — раунд 2: реальные root causes (2026-05-20)

Прошлая запись винила «api-gateway principal-resolution аномалию, не воспроизводится
локально». Это было неверно — анализ CI-run `26170966411` (api-gateway + iam логи)
вскрыл **4 конкретных** бага, все исправлены:

1. **Fixture aborts at Account.Create** (доминантный — каскадил в 110/511/21).
   `AccountService.Create` enforces `RequireOwnerMatchesPrincipal` (KAC-122 CRIT-3):
   principal обязан == `owner_user_id`. `setup.sh` создавал все Account/Project/
   Binding/SA/Network под `JWT_BOOTSTRAP` — BOOT≠AAA → `code 7` на первом
   `Account.Create`, фикстура падала, newman бежал по полу-засеянному стенду
   (пустые `{{accountAId}}` → все кейсы fail). Fix: каждая мутация фикстуры идёт
   под JWT владельца scope (account A → AAA, B → AAB; owner каскадит editor/admin
   на все project/SA аккаунта через FGA-иерархию). → kacho-workspace `688331e`.
2. **Permission catalog missing SAKeyService** (~21 authz-sa-apitoken fail).
   `protoc-gen-kacho-permissions` строит каталог по транзитивному import-closure
   `permissions_catalog_root.proto`; closure per-file → `sa_key_service.proto` (и
   federation/trust_policy) никто не импортировал → их RPC не в каталоге →
   api-gateway authz-mw fail-close `catalog: no entry → 403`. Fix: добавлены 3
   import'а в catalog-root (264→273 entries); SAKeyService scope исправлен
   `account`→`iam_service_account` (был mis-annotated). → kacho-proto `94609cc` +
   kacho-api-gateway `577f654` (embed sync).
3. **Phantom `authz-deny-rerun` suite** (511 fail-призрак). `tests/newman/out/`
   был committed в git со stale `authz-deny-rerun.json` от старого локального
   прогона; CI-checkout его восстанавливал, `run.sh` summary-loop по `out/*.json`
   считал призрак. Fix: `git rm --cached out/` + `.gitignore` + `run.sh` делает
   `rm -rf out` перед прогоном. → kacho-iam `b524217`.
4. **config inconsistency** — kacho-iam логировал `extapi.openfga.enabled=false`
   хотя FGA-клиент wired. Fix: `enabled: true` в values.dev + configmap. →
   kacho-deploy `ebbf96d`.

## newman-e2e round-3 — stale role-ids (RC: fixture data drift)

После round-2 fix'ов newman-e2e всё ещё падал (run `26178171977`: authz-deny
16 fail, authz-sa-apitoken 9 fail — все `no path` / PermissionDenied на
binding-производных Check'ах).

**Root cause** — НЕ panic, НЕ FGA-wiring. Фикстура и newman-кейсы хардкодили
legacy role-id'ы `rol00000000000000<tail>` из миграции `0001`. Миграция
`0008_role_catalog_kac122.sql` **DELETE'ит** старый каталог и пере-сидит его
детерминированными id `rol`+substr(md5(<name>),1,17). Любой
`AccessBinding.Create` / `Invite` с несуществующим `role_id` падал в worker'е
`FAILED_PRECONDITION Role <id> not found` (FK `access_bindings_role_fk`) →
binding не создавался → FGA grant-tuple не писался → каждый
binding-производный authz Check возвращал `no path`. AAA на своём account-A
проходил (owner-tuple), ломались только grant-производные (INV, SA-A).

**Fix** — замена на post-0008 global wildcard роли через именованные
константы (`admin`=`rol21232f297a57a5a74`, `view`=`rol1bda80f2be4d3658e`,
`edit`=`rolde95b43bceeb4b998`): kacho-iam `5752b58` (authz-deny.py /
authz-sa-apitoken.py), kacho-workspace `4100ae8` (tests/authz-fixtures/setup.sh).
