## Что и зачем

Все Kachō-сервисы, производящие Docker-образ, должны собирать multi-arch
(amd64 + arm64) образы и публиковать их в DockerHub. Раньше образы собирались
только локально (`kind load docker-image kacho-*:dev`), не было опубликованных
артефактов для production-деплоя.

Добавлен workflow `docker-build.yml` в 5 репозиториев (kacho-iam, kacho-vpc,
kacho-compute, kacho-api-gateway, kacho-ui) — паттерн адаптирован из
`PRO-Robotech/sgroups` / netguard reference. 3 job'а: amd64 (ubuntu-latest) +
arm64 (self-hosted) + multiarch manifest push. Secrets `DOCKERHUB_USERNAME` /
`DOCKERHUB_TOKEN`. Триггеры — push в `main` / `KAC-*` / тег `v*`.

Polyrepo-специфика: Go-сервисы (iam/vpc/compute/api-gateway) — Dockerfile делает
`COPY kacho-corelib` / `COPY kacho-proto`, поэтому workflow чекаутит siblings в
один каталог (build context = workspace-dir). kacho-ui — self-contained
(context = `.`).

kacho-deploy: umbrella Helm values переключены на DockerHub registry —
`global.imageRegistry` / `global.imageTag` (configurable); per-service `image:`
overrides в `values.yaml` (DockerHub default), `values.dev.yaml` (local `:dev`
образы для kind), `values.prod.yaml` (DockerHub + `${KACHO_IMAGE_TAG}` envsubst).

docker-build.yml дополняет существующие `ci.yaml` (тест/линт), не заменяет.
