## Что и зачем

Полная перестройка IAM Kachō в production-ready состоянии на основе iam-architect skill: ORY Kratos+Hydra AuthN, OpenFGA AuthZ с Conditions, OPA guardrails, Workload Identity Federation (для external CI/CD M2M), SCIM/SAML enterprise SSO, PIM/JIT elevation, break-glass, CAEP push pipeline.

**Out of scope (отдельные future эпики):**
- **Audit-infra**: Kafka + ClickHouse + S3 + HSM + SIEM + Merkle. На MVP — минимум `slog` + `audit_outbox` table-skeleton.
- **In-cluster Workload Identity + service mesh mTLS** (SPIFFE/SPIRE + Cilium/Linkerd). На MVP service-to-service auth = K8s NetworkPolicy + Internal listener + end-user principal forwarding в gRPC metadata (сохраняется KAC-108 модель).

## Decision Log (full в design doc; здесь high-level)

- D-1: AuthN = ORY Kratos + Hydra (Zitadel out)
- D-2: AuthZ = OpenFGA (Keto/Cedar/SpiceDB out for now)
- D-3: OPA sidecar для org-wide deny
- D-4: Hierarchy = cluster → org (optional) → account → project → resource
- D-5: Custom Role multi-scope
- D-6: DPoP-bound JWT
- D-7: WebAuthn/Passkey-first
- D-8: Workload Identity Federation (RFC 8693)
- D-9: SCIM 2.0
- D-10: SAML 2.0 через Jackson
- D-11: AccessBinding.status PENDING/ACTIVE/REVOKED
- D-12: Conditions overlay (CEL-like)
- D-13: PIM/JIT для admin
- D-14: Break-glass 2-person + 2h TTL
- D-15: CAEP push ≤10s revoke
- D-16: SPIFFE/SPIRE + service mesh — **OUT OF SCOPE** (future epic); MVP = K8s NetworkPolicy + end-user principal forwarding
- D-17: Cluster:singleton OpenFGA-object
- D-18: Audit MINIMUM (slog + outbox skeleton)
- D-19: Greenfield migration
