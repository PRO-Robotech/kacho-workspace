## Что и зачем

`values.dev.yaml` имел Kratos `selfservice.flows.<flow>.ui_url` + `default_browser_return_url` hardcoded на `http://localhost:28080/*`. Hydra `urls.consent/login/logout/post_logout_redirect` — то же. Это привязывало dev стенд к **одному** port-forward порту (:28080).

Когда пользователь делает `kubectl port-forward svc/ui 9091:8080` (другой порт), login flow Kratos редиректит браузер на :28080 → dead URL → cookie не ставится → API 401 → cascader не рендерится → tile clicks падают в edge-cases.

## Решение

Все browser-redirect URLs → relative paths:
- Kratos: `ui_url: /login` (etc.), `default_browser_return_url: /`, `base_url public: /.ory/kratos/public/`
- Hydra: `urls.consent/login/logout/error/post_logout_redirect → "/..."`, `urls.self.public/issuer → /.ory/hydra/public/`
- Whitelists: `allowed_return_urls` + CORS `allowed_origins` → wildcard `localhost:*`

Браузер сам резолвит relative paths к current origin — flow работает на любом порту.
