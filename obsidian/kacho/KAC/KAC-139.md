## Что и зачем

**Discovery**: middleware code (`authz.go:317-414`) уже fail-closed-correct since KAC-127.
Root cause — `values.prod.yaml` api-gateway block lacks `authz:` → chart default `{}` → middleware no-op в prod.

W1.3 ships **config + startup-validation**, не код middleware:
- **kacho-deploy/values.prod.yaml** — `authz.{enabled:true, failOpen:false}` + `authn.mode=production-strict` + `KACHO_APP_ENV=prod`
- **kacho-api-gateway/cmd/main.go** — `validateProductionAuthzConfig` fails-startup если `KACHO_APP_ENV=prod|staging` + (authz.enabled=false OR failOpen=true)
- **kacho-iam newman case** `AUTHZ-FAILCLOSED-OPENFGA-DOWN` (RED — orchestration deferred)
