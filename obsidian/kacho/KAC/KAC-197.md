# KAC-197: scope-extractor verb-suffix fix

**Status**: test (deployed на стенд, PR open)
**Type**: fix
**Repos**: kacho-api-gateway
**PR**: PRO-Robotech/kacho-api-gateway#43

## Что и зачем

Все мутирующие RPC с suffix-action на path-param ресурсе
(`/vpc/v1/subnets/{id}:add-cidr-blocks`, `:remove-cidr-blocks`, `:move`,
`:relocate`, IAM `:activate`/`:cancel`/`:approve` и т.д.) возвращали 403
PermissionDenied с reason `no path: unscoped resource`, даже для
cluster system_admin.

Bug в `internal/middleware/resource_extractor.go::extractByPathTemplate` —
проверял `t[len(t)-1] == '}'`, но в template `{subnet_id}:add-cidr-blocks`
последний символ `s` (verb tail). Placeholder не распознавался →
wildcard `vpc_subnet:*` → FGA Check no-path → 403.

## Fix

`splitVerbSuffix` отщепляет `{head}:verb` до placeholder-check. Verb
сегмента template и request path должны совпадать verbatim. Поддерживает
оба варианта: `{id}:verb` и `seg:verb`.

## Verify

- RED→GREEN: `TestResourceExtractor_FromHTTP_PathTemplate_VerbSuffix`
  (3 sub-cases — vpc-add-cidr-blocks / vpc-move / iam-activate) —
  все падали с `expected <id>, actual *`, все прошли после fix.
- E2E на стенде (docker.io/prorobotech/kacho-api-gateway:KAC-197-37d87c3):
  `POST /vpc/v1/subnets/<id>:add-cidr-blocks` → 200 Operation, done=true,
  response.v4CidrBlocks включает новый CIDR.
