## Что и зачем

Большой архитектурный shift IAM модели:
- **User per-Account**: один Kratos identity = N User-rows (по одному per Account куда invited). Сейчас был global User.
- **Invite-flow**: admin Account'а приглашает по email; backend создаёт User-stub status=PENDING; email НЕ отправляется (нет SMTP) — admin копирует magic-link вручную ИЛИ user сам идёт на /registration с тем же email и Kratos матчит. При first login `UpsertFromIdentity` активирует все PENDING-rows этого identity.
- **UI Cascader для ролей**: 3-level path (модуль/ресурс/verb), KAC-122 catalog format `<module>.<resource>.<verb>` без `roles/` prefix.
- **Системные / Свои роли**: separate tabs (is_system filter).
- **Default-deny on User.List**: anonymous → empty; user → видит только Users в своих Accounts.

Acceptance gate: 4 rounds review (5+3+3 blockers выявлены и закрыты). DRAFT v4 APPROVED.
