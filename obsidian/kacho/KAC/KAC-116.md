# KAC-116: Ory follow-up — закрыть DoD#3, #4, #5 + tech-debt KAC-115

**Status**: done (5 PRs open, ready merge; 9/9 Playwright pass)
**Parent epic**: [[KAC-104]]
**Predecessor**: [[KAC-115]]
**YT**: https://prorobotech.youtrack.cloud/issue/KAC-116

## Что и зачем

KAC-115 деплоил Ory stack (Kratos+Hydra+Keto+pg-*+kratos-selfservice-ui+MailHog),
но AuthZ-pipeline ещё ходил в OpenFGA, а auth — в Zitadel JWT. KAC-116 закрывает
этот tech-debt: Keto + Kratos session становятся primary path. DoD#3, #4, #5
теперь покрыты Playwright.

## Изменения

### Backend

- **kacho-iam (`#9`)**:
  - `internal/clients/keto_client.go` (new) — REST client Keto Read (4466) +
    Write (4467). Реализует `OpenFGAClient` interface (drop-in).
  - `internal/apps/kacho/api/internal_iam/handler.go` — `Check` RPC
    делегирует в OpenFGAClient.Check (=KetoClient при env=keto). Раньше
    Unimplemented → vpc/compute Check-interceptor возвращал fail-closed.
  - `cmd/kacho-iam/main.go` — `buildAuthZClient` env-switch (keto default;
    openfga legacy). Shared instance для Check + drainer.
  - Env: `KACHO_IAM_AUTHZ_BACKEND` / `KACHO_IAM_KETO_READ_URL` / `_WRITE_URL`.

- **kacho-api-gateway (`#15`)**:
  - `internal/middleware/kratos_session.go` (new) — `KratosClient.Whoami`
    с TTL=30s positive cache, 5s negative. JSON-deser Kratos session shape.
  - `internal/middleware/auth.go` — `WithKratos`; HTTP middleware:
    cookie ory_kratos_session → whoami → SubjectLookup → X-Kacho-Principal-*
    headers. JWT-path остаётся как fallback.
  - `internal/middleware/oidc_auth.go` — `OIDCHandler.Me` читает Kratos
    session-first; legacy Zitadel userinfo — fallback.
  - `internal/clients/iam_subject_client.go` — `LookupOrUpsertFromKratos`:
    при NotFound делает `InternalUserService.UpsertFromIdentity` (email +
    display_name из Kratos traits) + 5×200ms retry. Решает «new Kratos user
    не имеет User mirror в kacho-iam».
  - Env: `KACHO_API_GATEWAY_KRATOS_PUBLIC_URL`.

### Frontend (kacho-ui `#42`)

- `src/api/auth.ts` — `authApi.login()` → `/login` (Kratos self-service),
  убран Zitadel OAuth2 redirect (отдавал 404 в Ory stack).
- `src/contexts/AuthContext.tsx` / `src/components/auth/UserMenu.tsx` /
  `src/pages/auth/Logout.tsx` — logout стал sync (full-page nav на
  Kratos `/self-service/logout/browser`).
- `src/pages/auth/AuthCallback.tsx` — упрощён до landing'а для `?error=`.
- `deploy/default.conf.template` — nginx asset fallback: любые `.css/.js/.svg/.woff`
  на root или `/assets/*` пытаются сначала локально (kacho-ui dist/), при 404 →
  fallback в `kratos-selfservice-ui:3000`. Решает 404 на `/style.css`,
  `/theme.css`, `/main.css`, `/auth-layout.css`, `/ory-small.svg`,
  `/assets/inter-font.css` (Kratos UI pages их шлют из своего static-tree).
- Chart bump 1.4.0.

### Tests (kacho-test `#3`)

- `playwright/iam-userflow.spec.ts` — 6 → 9 тестов. Новые:
  - **DoD#4** — `/iam/v1/auth/me` с Kratos cookie → user.email matches.
  - **DoD#3** — IAM CRUD: List Roles (≥12 system) + List Accounts.
  - **DoD#5** — реактивность: `/iam/v1/roles` latency <2s (Keto pipeline);
    `/iam/v1/auth/me` hot <3s.
- `test.describe.serial` — registration test заполняет sharedCookieHeader,

### Deploy (kacho-deploy `#33`)
