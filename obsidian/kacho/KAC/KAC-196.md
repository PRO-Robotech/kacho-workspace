# KAC-196 — Cluster RBAC admin tooling

**Status**: in-progress · **Type**: feature · **YT**: [KAC-196](https://prorobotech.youtrack.cloud/issue/KAC-196)

## Что и зачем

Перевод управления «cluster admin» (root-grant на singleton `cluster_kacho_root`) из ручного `kubectl exec` / `openfga write` в UI-flow. Acceptance APPROVED в Round 2 (17 GWT, 13 Decisions, D-11 gate=`admin` computed alias → emergency_admin recovery работает, D-12 Revoke NOT idempotent → 404, D-13 OpenFGA outage = fail-fast Operation).

Текущее cluster_admin_grants хранится в `kacho_iam.cluster_admin_grants` (migration 0011, KAC-127); RPC handlers — Phase 2 этого тикета. **Без новых миграций** (acceptance §5.2).

## Архитектура

`InternalClusterService` (kacho-iam:9091) → api-gateway internal mux → `/iam/v1/internal/cluster/admins`. Single TX = INSERT `cluster_admin_grants` + INSERT `fga_outbox` + INSERT `audit_outbox` → return Operation. OpenFGA tuple emit через существующий drainer. Gate — FGA Check на `admin@cluster_kacho_root` (D-11 computed alias).

UI: `/system/cluster/admins` (таблица + Grant modal с email-search + Revoke per-row, disabled когда self или last).
