# KAC-196 — InternalClusterService cluster admin RBAC

**Status**: in-progress · **Type**: feature · **YT**: [KAC-196](https://prorobotech.youtrack.cloud/issue/KAC-196)

## Что и зачем

Перевод управления cluster admin (`system_admin@cluster:cluster_kacho_root`) из ручного `kubectl exec` / `openfga write` в UI flow. InternalClusterService с 4 RPC (`Get / GrantAdmin / RevokeAdmin / ListAdmins`), persistent audit trail через DB `cluster_admin_grants`, gate = `admin` computed alias (system_admin OR emergency_admin per D-11 — emergency_admin recovery работает).

Acceptance APPROVED Round 2 (17 GWT, 13 Decisions). Без новых миграций (cluster_admin_grants уже в squashed baseline 0001).

## Decisions reflected

- **D-2** SUBJECT_TYPE_USER only (handler rejects SERVICE_ACCOUNT)
- **D-4** idempotent Grant via ON CONFLICT + Reactivate (re-grant after revoke)
- **D-5** self-revoke guard (CAS WHERE `subject_id != $principal`)
- **D-6** last-admin guard (CAS WHERE `count(*) WHERE granted_until IS NULL > 1`)
- **D-9** user existence sync через `UserExistenceChecker`
- **D-11** gate = `admin` computed alias (emergency_admin recovery works)
- **D-12** Revoke NOT idempotent — 0 rows → NotFound
- **D-13** OpenFGA outage — repo-level atomic-emit-in-TX (DB row persists, drainer eventually pushes)
