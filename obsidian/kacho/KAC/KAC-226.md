# KAC-226: UI NLB список пуст — payloadKey load_balancers vs network_load_balancers

**Status**: done
**Type**: fix
**Repos**: kacho-ui
**PRs**: PRO-Robotech/kacho-ui#66 (merged)
**YT**: https://prorobotech.youtrack.cloud/issue/KAC-226

## Что и зачем

Registry-спека `load-balancers` имела `payloadKey: "load_balancers"`, тогда как proto `ListNetworkLoadBalancersResponse` несёт repeated-поле `network_load_balancers` (на проводе `networkLoadBalancers` → `camelToSnake` → `network_load_balancers`). `ResourceListPage.tsx:83` читает `data[spec.payloadKey]` → undefined → **список балансировщиков всегда пуст**, хотя backend вернул NLB. Та же ошибка в дашборд-плашке (`service-modules.tsx`).

Найдено при ручном тестировании стенда. `listeners` / `target_groups` payloadKey — корректны.
