## Что и зачем

Итерация 4 ролевой модели — закрытие 5 разрывов, найденных user'ом в админ-флоу:

1. **Group handler не имел Check** — anonymous видел Group через `/iam/v1/groups`. Default-deny через Keto: `viewer` на `account:<id>` для Get/List/ListMembers; `admin` для Create/Update/Delete/AddMember/RemoveMember. List без `account_id` → empty (если не system-admin).
2. **UI не отправлял `/iam/v1/accessBindings:listBySubject`** для текущего user'а — AccessBindingsPage по умолчанию открывалась в режиме `byResource` и `resId=""`. Добавлен top-блок **«Мои AccessBinding'и»** с auto-query через `useAuth()`.
3. **AccountCrumb / ProjectCrumb селекторы в шапке не работали** — `DropdownRow` использовал raw `<button>` внутри `DropdownMenu.Content` (Radix не закрывает меню → клик визуально «не срабатывал»). Заменено на `DropdownMenu.Item` с `onSelect`+`preventDefault`. Дополнительно: ProjectCrumb передавал `?accountId=...` (camelCase), backend ожидает `?account_id=...` — пофикшен query param.
4. **Admin (@prorobotech.ru) не видел «Регионы/Пулы/Зоны»** — `permissions=["*","admin"]` для system-admin резолвится в api-gateway через `WithAdminChecker` → `IAMSubjectClient.IsSystemAdmin` → Keto `Check(user:<id>, admin, kacho_system:root)`. UI читает `hasPermission('*')` через `useAuth()`. Backend код был готов после предыдущей итерации, требовался деплой.
5. (deferred → KAC-124) Полное удаление `kacho-resource-manager` — большой рефакторинг с acceptance-doc первым.
