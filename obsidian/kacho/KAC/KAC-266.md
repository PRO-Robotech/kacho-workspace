# KAC-266: contract-removal — Move / NIC attach-detach / AddressPool override+selector / no auto-NIC

## Что и зачем

Срезаем неиспользуемую/переусложнённую часть контракта (proto Wave-1 + сервисы Wave-2). Удалённые RPC:

- **Move** — vpc `Network`/`Subnet`/`Address`/`RouteTable`/`SecurityGroup`/`Gateway` + compute `Instance`/`Disk` + iam `Project`. Cross-folder/cross-project move убран; owner (`project_id`/`account_id`) неизменяем после Create.
- **Subnet.Relocate** — удалён (cross-zone relocate). *(KEEP: compute `Instance`/`Disk` Relocate, NLB Move.)*
- **NIC attach/detach** — vpc `NetworkInterfaceService.AttachToInstance` / `DetachFromInstance`. `used_by_id` больше не выставляется этими RPC.
- **AddressPool** — `BindAsAddressOverride` / `UnbindAddressOverride` + `Check` + `ExplainResolution`. *(KEEP: `BindAsNetworkDefault` / `UnbindNetworkDefault`, AddressPool CRUD.)*
- **InternalCloudService** — удалён **целиком** (`SetPoolSelector` / `GetPoolSelector` / `UnsetPoolSelector`).
- **IPAM-cascade** упрощён до `network_default` → `zone_default` → `global_default` (шаги override + cloud-selector сняты).

**Compute: инстанс создаётся БЕЗ авто-NIC** — `materializeNICs` удалён; `network_interface_specs` в `CreateInstanceRequest` игнорируется; ребро compute→vpc NIC-attach разорвано. compute proper rework (как NIC будет привязываться) — **deferred**.

**KEEP**: публичный vpc `NetworkInterface` CRUD; AddressPool CRUD + `BindAsNetworkDefault`; compute `Instance`/`Disk` `Relocate`; NLB `Move`; compute→vpc Address-IPAM (NAT) edge.
