# KAC-242 — kacho-ui: действия из таба «Обзор» в шапку detail-страницы

**Status**: test (PR #69, задеплоено на kind-стенд) · **Type**: feature(UX) · **Repos**: kacho-ui · **Parent**: [[KAC-239]]

## Что и зачем

Действия ресурса (`DetailOverviewActions`: «Редактировать» + ⋮-меню «Удалить» + ext-actions)
жили в `SectionHeader` таба «Обзор». Перенесены в **шапку страницы** — правый слот верхнего
хедера (`PageHeaderSlot` `useHeaderRight`, рядом с хлебными крошками). До этого правый слот хедера
на detail-страницах был пуст (ResourceShell ставил только breadcrumb).

## Как сделано

- `ResourceShell`: `useHeaderRight(headerActions)`, где `headerActions` = `<DetailOverviewActions/>`,
  мемоизирован, рендерится **только во view-режиме** (`!mode`) — при `edit`/`child-create` форма уже
  в зоне 3, кнопки скрыты (`null`). React Query structural-sharing держит `data`-ref стабильным → нет
  5s-перерендера хедера.
- Таб «Обзор» `SectionHeader` стал title-only (`<SectionHeader title="Обзор" />`), контент = Descriptions.
- `DetailOverviewActions` — обновлён doc-коммент (теперь шапка, не таб «Обзор»).
