# KAC-242 — kacho-ui: действия из таба «Обзор» в шапку detail-страницы

**Status**: test (PR #69, задеплоено на kind-стенд) · **Type**: feature(UX) · **Repos**: kacho-ui · **Parent**: [[KAC-239]]

## Что и зачем

Действия detail-страницы перенесены в **шапку** (правый слот верхнего хедера, `PageHeaderSlot`
`useHeaderRight`, рядом с хлебными крошками — раньше был пуст). **Ключевое: действия КОНТЕКСТНЫЕ
по активному табу**, а не глобальные на всех табах (первый cut `a5f1e47` вешал Редактировать/Удалить
ресурса на ВСЕ табы — неверно, исправлено `808c21a`).

## Как сделано (per-tab, `808c21a`)

`ResourceShell`: `useHeaderRight(headerActions)`, где `headerActions` зависит от **активного таба**
(берётся из URL до early-return, мемоизируется):
- **«Обзор»** → `<DetailOverviewActions/>` (Редактировать + ⋮Удалить ресурса + ext-actions).
- **related-child таб** (подсети / таблицы маршрутизации / группы безопасности / …) →
  «Создать <child>» (кнопка поднята из `RelatedTable` `SectionHeader` в шапку); удаление ребёнка —
  per-row в таблице (`RowActionsMenu`).
- **прочие табы** (Операции / JSON / ext) → нет.
- Скрыто в `edit`/`child-create` (форма в зоне 3). React Query structural-sharing → `data`-ref
  стабилен, нет 5s-перерендера.

`RelatedTable`: «Создать <child>» убрана из `SectionHeader` (теперь в шапке), поиск+колонки остались.
Таб «Обзор» `SectionHeader` — title-only.

> [!note] SG «привязать» (bind existing SG) из требования — **не существует** в продукте (нет
> associate-RPC). Реализован «Создать SG» + per-row delete; «привязать» — отдельная фича (backend).
