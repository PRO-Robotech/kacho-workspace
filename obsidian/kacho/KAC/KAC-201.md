# KAC-201: Internal-tier authz hardening (epic)

**Status**: in-progress (acceptance-author работает; coding gate — APPROVED acceptance, workspace CLAUDE.md §запрет 1)
**Type**: epic
**YT**: https://prorobotech.youtrack.cloud/issue/KAC-201

## Discovered

Все **44** метода `Internal.*` сервисов в `permission_catalog.json` имеют пустые `permission` и `required_relation` → api-gateway authz-mw passthrough → любой authenticated user через ingress может вызвать любой Internal-метод.

User-reported reproduction: regular user → `GET /vpc/v1/addressPools` → 200 + listing default-pools (должно быть 403; AddressPool kacho-only admin per kacho-vpc/CLAUDE.md §16).

## Scope — 11 сервисов, 3 категории

**Категория A — Admin-only** (`system_admin@cluster:cluster_kacho_root` FGA):
- `InternalAddressPoolService.*` (13 methods)
- `InternalCloudService.{Set,Unset,Get}PoolSelector`
- `InternalNetworkService.GetNetwork` (vpn_id sensitivity)
- `InternalRegionService.{Create,Update,Delete}`
- `InternalZoneService.{Create,Update,Delete}`
- `InternalDiskTypeService.{Create,Update,Delete}`

**Категория B — Inter-service-only** (service-account subject-type required):
- `InternalAddressService.Allocate*` / `Free*` (kacho-vpc inline)
- `InternalIAMService.{Check,LookupSubject,WriteCreatorTuple}`
- `InternalUserService.UpsertFromIdentity`
- `InternalResourceLifecycleService.Subscribe`

**Категория C — Public-read** (cluster.viewer cascade):
- `InternalRegionService.{Get,List}`
- `InternalZoneService.{Get,List}`
- `InternalDiskTypeService.{Get,List}`
- `InternalWatchService.Watch`

## Workplan (subtasks — будут созданы после APPROVED acceptance)

1. FGA model + seed (cluster relations: system_admin / viewer / service_account)
2. Catalog migration — все 44 method'а получают корректный permission/relation/scope
3. Service-account distinct subject-type в kacho-corelib auth interceptor
4. api-gateway authz-mw fail-closed на empty entry для `.Internal*` paths
5. TLS-listener filter — defence-in-depth для external endpoint (workspace CLAUDE.md §6)
6. Newman regression — per-method: regular-user → 403, admin → 200, SA → 200 (для B)
7. Vault trail (edges/* + resources/* per-category)
