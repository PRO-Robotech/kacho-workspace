## Что и зачем

Унификация всего UI под единый 3-зонный лайаут: **сайдбар | табы (Обзор/JSON/Связанные) | контент-или-форма**. Формы — расширение страницы (правая зона), не модалки. Уникальный URI на каждую страницу. Registry-driven (`ResourceShell` + `FormPanel`) + слоты для спец-кейсов. Разворачивает `kacho-ui/CLAUDE.md §3` (модалки → панели).

## Ключевые решения (brainstorming)

- Форма = всегда правая зона: список → `sidebar|таблица|форма` (`/create`); detail → `sidebar|табы|форма` (форма заменяет инфо; `/edit`, `/<child>/create`).
- Таб «Связанные» = список встроенных `<ResourceTable>` (тот же компонент, что на списке) + «Создать».
- Эталон: **VPC Network** (удаляем `NetworkDetailPage`).

## Прогресс реализации (ветка `KAC-232`, не в main)

> [!note] Заказчик выбрал «строй сразу, оцениваю визуально» — итерации деплоятся на стенд (`reload-svc SVC=ui`), acceptance/тесты — перед PR в main.

**Готово (KAC-232 framework + KAC-234 VPC):**
- `ResourceShell` — registry-driven: detailBase из pathname (любая вложенность/модуль), `spec.related/docs/emptyState`, merge `DETAIL_EXTENSIONS` (overviewExtra/overviewBelow/extraTabs/headerActions/hideOperations/title), `internalGetPath`→jsonint, форма-панель через `InlineResourceForm`.
- Компоненты: `InlineResourceForm` (общий диспетч форм; `ResourceFormModal` делегирует), `SectionHeader`, `TableToolbar` (поиск+gear колонок), `ResourceEmptyState` (welcome, переиспользован list+related), `resource-detail-extensions` (доменный контент per-spec).
- VPC: все 7 ресурсов на ResourceShell; домен сохранён (SG ingress/egress-табы +«Добавить правило», RouteTable static_routes отдельной таблицей в Обзоре, Subnet CIDR-теги копируемые, Address used_by→ReferrerLink, NIC адреса/SG тегами). Flat-URL + breadcrumb.
- Модалки упразднены у **VPC + admin** (panelForms): create→`/create`-страница, edit→панель/страница. Compute/NLB/IAM — на модалках до их раскатки.
- Ссылки на ресурсы: единый вид **иконка+имя** (`RefNameLink`); `ReferrerLink` маппит referrer.type→spec (вкл. network_interface).
- Операции — новое сверху. Пустые таблицы (list+related) → welcome.

**Осталось:** [[KAC-235]] Compute, [[KAC-237]] NLB, [[KAC-236]] IAM (раскатка + их модалки→панели), [[KAC-238]] cleanup (снос `GlobalResourceFormModal`/§3, vitest RED→GREEN, PR в main), durable-DNS fix (ndots в helm-чарт kacho-deploy).
