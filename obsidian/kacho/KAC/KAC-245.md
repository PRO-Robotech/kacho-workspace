# KAC-245 — UI: Name+ID первые две колонки во всех таблицах по умолчанию

**Status**: test (ui#71 merged, деплой kind) · **Type**: feature (UX) · **Repos**: kacho-ui

## Что и зачем

Пользователь: «все таблицы должны иметь по дефолту первые две колонки name и ID».
Было неоднородно: VPC/compute — Name,ID первые; IAM (accounts/projects/service-accounts)
и security-groups — Name первая, ID в конце; regions/zones — ID первая, Name дальше.

## Решение (одна точка, generic)

`src/lib/spec-columns.tsx` → новая `reorderNameIdFirst(columns)`, вызывается в
`buildSpecColumns()`. Извлекает колонки `path==="name"`/`"id"` (где бы ни стояли) и
ставит первыми (Name, затем ID), **сохраняя сами объекты** → кастомные render
(CopyableName/CopyableId) не теряются; остальные — в исходном порядке. Нет name-колонки
(disk-types/compute-zones) → id остаётся первым (graceful). Идемпотентно.

Применяется автоматически и к `ResourceListPage` (список), и к `ResourceShell` RelatedTable
(дочерние таблицы на detail) — обе зовут `buildSpecColumns`. Per-resource `columns` в
`resource-registry.tsx` НЕ трогали.
