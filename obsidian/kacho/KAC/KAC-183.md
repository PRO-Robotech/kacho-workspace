## Что и зачем

CI run 26358993224 — 6 newman failures в `iam-compliance-report` suite. Триаж (см. `docs/specs/newman-80-failures-finding.md` v2, F1):

- `helm/umbrella/values.dev.yaml:813-851` объявляет `KACHO_IAM_COMPLIANCE__S3__ENDPOINT=http://minio-dev:9000` + `compliance.s3` блок.
- `helm/umbrella/templates/` **не имел** `minio*` шаблона.
- → DNS `minio-dev` неразрешим → `kacho-iam s3_report_store::doPut` retries timeout → `ComplianceReportService.GenerateAccessReport` уходит в `status=FAILED`.

Фикс — добавил `templates/minio-dev.yaml`:
- Deployment (1 replica, emptyDir, `minio/minio:RELEASE.2024-12-18T13-15-44Z`, console на :9001)
- Service ClusterIP (порты 9000 S3, 9001 console) — соответствует `KACHO_IAM_COMPLIANCE__S3__ENDPOINT`
- Init Job (`minio/mc`, post-install/upgrade hook) — ждёт API + создаёт bucket `kacho-compliance-reports` + `anonymous set none`
- Всё gated на `.Values.minioDev.enabled`

`values.dev.yaml`: `minioDev.enabled=true` + дефолтные `minioadmin/minioadmin`.
`values.prod.yaml`: `minioDev.enabled=false` — prod использует external managed S3/MinIO с creds из Vault/SealedSecret.
