## Cat-C: что починили

### iam-jit-pending (46/46, 0 failures)
- **SQLSTATE 42P18 fix**: `scopePredicate` вызывался с `base=3` → генерировал `$4` для scope_id,
  оставляя `$3` (scope_type из `scopeArgs`) unreferenced → postgres не мог вывести тип параметра.
  Фикс: `base=2`, args `[start, end, scope_id, cursor]` без scope_type.
- **JITWindowConditionID=""**: `cnd_jit_window` row не pre-seeded → FK violation при approve.
  Обнулили; expiry enforced by `expires_at` колонкой.
- **Approve/Deny handler**: возвращают `p.OperationID` (оригинальный iop из DB, уже `done=true`).
- **GetJitPending scope-check**: NOT_FOUND для вызывающих не являющихся requester/approver.
- **CreatedAt/ExpiresAt**: truncate to second (YC-style).
- **KNOWN-GAP**: SCOPED test — `ListJitPendingRequest` proto не имеет `account_id`; фильтрация по
  `?accountId=` игнорируется grpc-gateway. Follow-up proto change.
- **reseed_jit()** в run.sh: идемпотентный сброс seed rows в PENDING перед каждым прогоном суиты;
  удаляет materialised bindings от предыдущего approve.

### iam-compliance-report (49/49, 4 intentional RED #37)
- **SCOPE-NOTFOUND**: FGA проверяет scope перед service-layer → 403 для несуществующего scope_id.
  Тест принимает 200/403/404.
- **opId stale**: `generate-missing-scope` теперь делает `pm.environment.unset('opId')` при 403/404,
  чтобы `poll-missing-scope` не переиспользовал opId из предыдущего теста.
- **cmr regex**: `/^cmr[_a-z0-9]+$/` (underscore в id cmr_xxx).
- **poll_operation_until_done**: max retries 8→20 (compliance worker 1s cadence).
- **S3 config**: compliance.s3 YAML section в configmap.yaml (MinIO dev endpoint). Viper env binding
  для hyphenated fields (`path-style`) ненадёжен → YAML path.
- **Helm values**: `kacho-iam.compliance.s3` + `GOVERNANCE__COMPLIANCE_REPORT_WORKER_INTERVAL_SECONDS=1`.
- **4 intentional RED #37**: GET/DL foreign-user scope-check — authz remediation finding #37.
