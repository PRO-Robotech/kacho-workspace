# KAC-251: kacho-proto — apisurface + openapi-filter

## Что и зачем

Сделать `kacho-proto` единственным источником истины публичной API-поверхности
(backbone эпика [[KAC-248]]):

1. Перенести allowlist (`AllowedMethods` + `IsAllowed` + `HasInternalSuffix`) из
   `kacho-api-gateway/internal/allowlist` в `kacho-proto` как пакет **`apisurface`**
   (handwritten, НЕ под `gen/`) — устраняет split-brain (api-gateway будет импортировать
   его в [[KAC-253]]).
2. Генерация публичного OpenAPI 3.1 из публичных proto: `protoc-gen-connect-openapi`
   (raw, отдельный шаблон `buf.gen.openapi.yaml`) → overlay curated-examples →
   `cmd/openapi-filter` (recursive WalkDir по 136 файлам, бакетинг по домену, фильтр по
   `apisurface`, **scope `components/schemas` до `$ref`-достижимых от выживших операций**,
   инжект `x-operation` из ext 87334) → commit `gen/openapi/<domain>.openapi.json` +
   `gen/openapi/_surface-snapshot.json`.
3. Drift-gate `verify-openapi` (`git diff --exit-code gen/openapi/`) в Makefile + CI по
   образцу существующего `verify-catalog` (permission_catalog.json).

## Ключевой нюанс (проверено реальным прогоном при планировании)

- connect-openapi эмитит 136 файлов **зеркальным деревом** (one-per-proto), не flat
  per-domain → фильтр на recursive WalkDir.
- Слепое объединение схем (896) затащило бы `vpn_id`/`sid` из dropped
  `NetworkInterfaceService` в vpc-спеку → **$ref-scope обязателен** (проверено: 191
  survivor, 0 Internal, 0 AddressPool, 0 infra-token в reachable-замыканиях).
- 6 `Internal*`-proto несут `google.api.http` → http-наличие НЕ дискриминатор →
  allowlist-фильтр обязателен.

## Finding: description-prose leak → решение A (sanitizer guard)

Ревью сгенерированного OpenAPI вскрыло: операции/схемы/поля чисты ($ref-scope убрал
`vpn_id`/`sid`-поля, 0 Internal/AddressPool-операций), но ~80 `description`-полей несут
internal-engineering прозу из proto-комментариев (`InternalIAMService`,
`vpc.InternalAddressService`, `AddressPool (см. CLAUDE.md §16)`, `KAC-127`, `design §`,
`OpenFGA`). Scoped-tripwire исполнителя это пропустил (грепал только example-блоки).

**Решение (A):** в `openapi-filter` добавлен **description-sanitizer guard-by-construction**
(детерминированно вырезает §9-токены + internal-noise из описаний, дропает пустые) +
tripwire расширен на **whole-file** (описания) с word-boundary. → публичный артефакт
§9-чист by construction. Durable fix источника (proto-комментарии + godoc) — отдельный
[[KAC-260]] (proto-api-reviewer, НЕ блокирует MVP).

## Решение по B-04 (OperationService.Cancel rendering) — ЗАФИКСИРОВАНО

**Default: snapshot-presence.** `OperationService.Cancel` присутствует в
`_surface-snapshot.json` (удовлетворяет B-05 + B-04 «distinct operation в нормативном
артефакте поверхности»); отдельный `operation.openapi.json` НЕ эмитится —
OperationService документируется общим разделом «Работа с API», не per-domain.
